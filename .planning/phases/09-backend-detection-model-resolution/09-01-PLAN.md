    const fs = require('fs');

    **Constants to export:**

    1. `VALID_BACKENDS` -- Array: `['claude', 'codex', 'gemini', 'opencode']`

    2. `DEFAULT_BACKEND_MODELS` -- Object per ARCHITECTURE.md:
       ```javascript
       {
         claude:   { opus: 'opus',           sonnet: 'sonnet',              haiku: 'haiku' },
         codex:    { opus: 'gpt-5.3-codex',  sonnet: 'gpt-5.3-codex-spark', haiku: 'gpt-5.3-codex-spark' },
         gemini:   { opus: 'gemini-3-pro',   sonnet: 'gemini-3-flash',      haiku: 'gemini-2.5-flash' },
         opencode: { opus: 'anthropic/claude-opus-4-5', sonnet: 'anthropic/claude-sonnet-4-5', haiku: 'anthropic/claude-haiku-4-5' },
       }
       ```

    3. `BACKEND_CAPABILITIES` -- Object per ARCHITECTURE.md:
       ```javascript
       {
         claude:   { subagents: true,  parallel: true,  teams: true,  hooks: true,  mcp: true  },
         codex:    { subagents: true,  parallel: true,  teams: false, hooks: false, mcp: true  },
         gemini:   { subagents: 'experimental', parallel: false, teams: false, hooks: true, mcp: true },
         opencode: { subagents: true,  parallel: true,  teams: false, hooks: true,  mcp: true  },
       }
       ```

    **Functions to implement:**

    4. `detectBackend(cwd)` -- Detection waterfall per multi-backend-detection.md Section 2:
       - Step 1: Load config from `cwd/.planning/config.json`. If `config.backend` is set AND is a valid backend, return it immediately. Use a lightweight config read (just parse the JSON and check the `backend` field) -- do NOT call `loadConfig()` from utils.js to avoid circular dependency risk. Just read the file directly with `fs.readFileSync` and `JSON.parse`.
       - Step 2: Check env vars in order: CLAUDE_CODE_* prefix (use `Object.keys(process.env).some(k => k.startsWith('CLAUDE_CODE_'))`), then CODEX_HOME/CODEX_THREAD_ID, then GEMINI_CLI_HOME, then OPENCODE.
       - Step 3: Check filesystem clues in cwd: `.claude-plugin/plugin.json`, `.codex/config.toml`, `.gemini/settings.json`, `opencode.json`.
       - Step 4: Default to `'claude'`.

       **IMPORTANT (from PITFALLS.md P5):** Do NOT use the `AGENT` env var for OpenCode detection -- it is too generic and may collide with other tools. Only use `OPENCODE`.

    5. `resolveBackendModel(backend, tier, config)` -- Model resolution per ARCHITECTURE.md:
       - If `config.backend_models?.[backend]?.[tier]` exists, return it (user override).
       - Otherwise return `DEFAULT_BACKEND_MODELS[backend]?.[tier]`.
       - If backend is unknown, fall back to claude mappings.
       - If tier is unknown, return undefined.

    6. `getBackendCapabilities(backend)` -- Simple lookup:
       - Return `BACKEND_CAPABILITIES[backend]` or `BACKEND_CAPABILITIES.claude` for unknown backends.

    **Module pattern:** Follow existing lib/*.js conventions:
    - JSDoc on all exported functions
    - `module.exports = { ... }` at the bottom
    - File header comment matching other lib files

    **Avoid:**
    - Do NOT import from `lib/utils.js` (avoid circular dependency since utils.js will later import from backend.js). Read config.json directly with fs.readFileSync.
    - Do NOT use the `AGENT` env var for OpenCode detection (P5 pitfall).
    - Do NOT use `GEMINI_API_KEY` for Gemini detection -- it can be set without Gemini CLI running.
