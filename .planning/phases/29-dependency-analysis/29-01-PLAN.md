    **Function 1: parseDependsOn(dependsOnStr)**
    - Input: raw string from `**Depends on:**` field in ROADMAP.md (e.g., "Phase 27, Phase 29", "Nothing", null)
    - Output: array of phase number strings (e.g., `['27', '29']`, `[]`)
    - Implementation:
      - Return `[]` for null, undefined, empty string
      - Return `[]` if string contains "nothing" (case-insensitive)
      - Use regex `/Phase\s+(\d+(?:\.\d+)?)/gi` to extract all phase number references
      - Return array of matched phase number strings

    **Function 2: buildDependencyGraph(phases)**
    - Input: array of objects with `{ number, name, depends_on }` (the depends_on is the raw string)
    - Output: `{ nodes: [{id, name}], edges: [{from, to}] }`
    - Implementation:
      - Map each phase to a node: `{ id: phase.number, name: phase.name }`
      - For each phase, call `parseDependsOn(phase.depends_on)` to get deps
      - For each dep, create edge: `{ from: dep, to: phase.number }` (dep must complete before phase)
      - Filter edges to only include deps whose `from` exists in the node set (ignore dangling refs)

    **Function 3: computeParallelGroups(graph)**
    - Input: `{ nodes, edges }` from buildDependencyGraph
    - Output: array of arrays (each inner array = set of phase IDs that can run together)
    - Implementation (Kahn's algorithm / topological sort by levels):
      - Build adjacency list and in-degree map from edges
      - Initialize: all nodes with in-degree 0 go into first group
      - Iteratively: remove current group's nodes, decrement in-degrees for their dependents, collect next group (new in-degree 0 nodes)
      - Continue until no nodes left (or cycle detected â€” but cycle detection is separate)
      - Sort phase IDs within each group for deterministic output
      - Return empty array for empty graph

    **Function 4: detectCycle(graph)**
    - Input: `{ nodes, edges }` from buildDependencyGraph
    - Output: null if acyclic, or array of phase number strings forming the cycle (e.g., `['27', '28', '27']`)
    - Implementation (DFS-based cycle detection):
      - Build adjacency list (forward edges: from -> [to])
      - Track node states: unvisited, visiting (in current DFS path), visited
      - For each unvisited node, run DFS
      - If DFS encounters a "visiting" node, reconstruct cycle path from the stack
      - Return null if all nodes complete DFS without finding a back-edge

    **Function 5: cmdPhaseAnalyzeDeps(cwd, raw)**
    - CLI command function following the established `cmd*` pattern
    - Implementation:
      1. Read ROADMAP.md from `path.join(cwd, '.planning', 'ROADMAP.md')`
      2. If not found, call `output({ error: 'ROADMAP.md not found' }, raw, '')` and return
      3. Strip shipped sections with `stripShippedSections`
      4. Parse phases using the same regex pattern as `cmdRoadmapAnalyze`: `/###\s*Phase\s+(\d+(?:\.\d+)?)\s*:\s*([^\n]+)/gi`
      5. For each phase, extract the `**Depends on:**` field from its section text
      6. Build graph with `buildDependencyGraph`
      7. Check for cycles with `detectCycle`
      8. If cycle found: output `{ error: 'Circular dependency detected', cycle_path, has_cycle: true, nodes, edges }`
      9. If no cycle: compute parallel groups, output `{ nodes, edges, parallel_groups, has_cycle: false, phase_count, group_count }`
    - Use `output()` for JSON and raw text output (matching codebase pattern)
