  <name>Task 1: Create GitHub Actions release workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
    Create `.github/workflows/release.yml` with the following configuration:

    **Trigger:**
    - `workflow_dispatch` only (manual trigger from GitHub UI)

    **Job: release**
    - `runs-on: ubuntu-latest`
    - Node.js version: 22 (latest LTS for release builds)
    - Steps:
      1. `actions/checkout@v4`
      2. `actions/setup-node@v4` with `node-version: 22`
      3. `npm ci` (or `npm install` if no lockfile)
      4. **Version consistency check** (shell script step):
         ```bash
         VERSION_FILE=$(cat VERSION)
         PLUGIN_VERSION=$(node -e "console.log(require('./.claude-plugin/plugin.json').version)")
         CHANGELOG_HAS=$(grep -c "\\[$VERSION_FILE\\]" CHANGELOG.md || true)
         echo "VERSION file: $VERSION_FILE"
         echo "plugin.json:  $PLUGIN_VERSION"
         echo "CHANGELOG:    $CHANGELOG_HAS match(es)"
         if [ "$VERSION_FILE" != "$PLUGIN_VERSION" ]; then
           echo "::error::VERSION ($VERSION_FILE) does not match plugin.json ($PLUGIN_VERSION)"
           exit 1
         fi
         if [ "$CHANGELOG_HAS" -eq 0 ]; then
           echo "::error::CHANGELOG.md has no entry for version $VERSION_FILE"
           exit 1
         fi
         echo "Version consistency: OK"
         ```
      5. **Run full test suite:** `npm test`
      6. **Extract release notes from CHANGELOG** (shell script step):
         ```bash
         VERSION=$(cat VERSION)
         # Extract the section for current version from CHANGELOG.md
         # Matches from "## [$VERSION]" to the next "## [" or end of file
         awk "/^## \\[$VERSION\\]/,/^## \\[/" CHANGELOG.md | head -n -1 > release-notes.md
         echo "--- Release Notes ---"
         cat release-notes.md
         ```
      7. **Create GitHub Release** using `softprops/action-gh-release@v2`:
         - `tag_name: v${{ steps.version.outputs.version }}` (read VERSION into output in step 4)
         - `body_path: release-notes.md`
         - `draft: true` (release as draft for human review before publishing)

    **Important details:**
    - The version consistency step should set an output for the version value: add `echo "version=$VERSION_FILE" >> $GITHUB_OUTPUT` and give the step an `id: version`
    - The release is created as a DRAFT so the maintainer can review before publishing
    - Use `permissions: contents: write` at the job level for release creation
    - The workflow should NOT create or push git tags -- the tag is expected to be created manually or by a separate process
    - Add `if: success()` on the release step to ensure it only runs if all prior steps pass
    - No npm publish step -- GRD is installed via git clone, not npm registry
  </action>
