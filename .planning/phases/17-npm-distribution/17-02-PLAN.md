<task type="auto">
  <name>Task 1: Implement cmdSetup in lib/commands.js and wire into CLI router</name>
  <files>lib/commands.js, bin/grd-tools.js</files>
  <action>
    **1. Add cmdSetup function to lib/commands.js:**

    The setup command helps users configure Claude Code to use the installed GRD package. It needs to:

    a. **Locate the GRD installation:**
       - Use `path.resolve(__dirname, '..')` to find the package root (lib/commands.js is in lib/, so one level up is the package root)
       - Verify `.claude-plugin/plugin.json` exists at that location
       - If not found, `error('GRD installation not found. Run this command from a valid GRD installation.')` and exit

    b. **Output plugin configuration info:**
       - Resolve the absolute path to the package root
       - Resolve the absolute path to `.claude-plugin/plugin.json`
       - Output JSON (or raw text) with:
         ```json
         {
           "package_root": "/absolute/path/to/grd-tools",
           "plugin_json": "/absolute/path/to/grd-tools/.claude-plugin/plugin.json",
           "instructions": "Add to Claude Code settings:\n  \"plugin_path\": \"/absolute/path/to/grd-tools/.claude-plugin\""
         }
         ```
       - In raw mode, output human-readable instructions:
         ```
         GRD plugin configured.
         Package root: /absolute/path/to/grd-tools
         Plugin config: /absolute/path/to/grd-tools/.claude-plugin/plugin.json

         To use with Claude Code, add this plugin path to your Claude Code configuration:
           /absolute/path/to/grd-tools/.claude-plugin
         ```

    c. **Function signature:** `function cmdSetup(cwd, raw)` — consistent with other cmd* functions.
       - Use `output()` from utils.js for JSON/raw output (already imported in commands.js).

    d. **Add cmdSetup to the module.exports object** at the bottom of commands.js.

    **2. Wire setup command into bin/grd-tools.js:**

    a. Add `cmdSetup` to the destructured import from `'../lib/commands'`:
       ```js
       const {
         cmdGenerateSlug,
         // ... existing imports ...
         cmdQualityAnalysis,
         cmdSetup,  // ADD THIS
       } = require('../lib/commands');
       ```

    b. Add case to the switch in `routeCommand()`, before the `default` case:
       ```js
       case 'setup':
         cmdSetup(cwd, raw);
         break;
       ```

    c. Update the usage string in the `if (!command)` error message to include `setup`.

    **Important:** Do NOT add `setup` as a subcommand of another command. It is a top-level command: `grd-tools setup`.
  </action>
  <verify>
    1. `node bin/grd-tools.js setup --raw` outputs plugin path information (Level 1: Sanity)
    2. `node bin/grd-tools.js setup` outputs valid JSON with package_root and plugin_json fields (Level 2: Proxy)
    3. Paths in output are absolute and point to existing files
  </verify>
  <done>
    `grd-tools setup` command is routed and implemented. It resolves the package root, locates .claude-plugin/plugin.json, and outputs the configuration path. Works in both JSON and raw output modes.
  </done>
