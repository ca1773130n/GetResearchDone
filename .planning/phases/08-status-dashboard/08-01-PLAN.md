    **1. cmdDashboard(cwd, raw):**

    Aggregates all milestones, phases, and plans into a tree structure.

    Data gathering:
    - Read ROADMAP.md, parse milestone headings (`## Milestone N: Name` or `## MN vX.Y: Name`) with Start/Target dates
    - Parse phase headings (`### Phase N: Name`) with Duration, Type
    - For each phase, read its directory in `.planning/phases/` to count plans, summaries, and determine status
    - Read STATE.md to get current position (active phase), blockers count, deferred validations count, decisions count

    JSON output structure:
    ```json
    {
      "milestones": [
        {
          "name": "Foundation",
          "version": "v0.0.5",
          "start": "2026-02-13",
          "target": "2026-02-17",
          "phases": [
            {
              "number": "1",
              "name": "Security Foundation",
              "type": "implement",
              "duration": "2d",
              "plans": 1,
              "summaries": 1,
              "status": "complete",
              "active": false
            }
          ],
          "progress_percent": 100
        }
      ],
      "summary": {
        "total_milestones": 4,
        "total_phases": 8,
        "total_plans": 14,
        "total_summaries": 14,
        "active_blockers": 0,
        "pending_deferred": 2,
        "total_decisions": 20
      }
    }
    ```

    TUI output (when not --raw): Render tree view using ui-brand.md patterns:

    Status symbols per ui-brand.md:
    - `✓` for complete (summaries >= plans > 0)
    - `◆` for in-progress (summaries > 0 but < plans)
