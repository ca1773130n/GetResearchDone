    Create `lib/worktree.js` following the established module pattern (see lib/phase.js for reference):

    **Module structure:**
    ```javascript
    /**
     * GRD Worktree — Git worktree lifecycle management for phase isolation
     *
     * Provides create, remove, list, and stale cleanup for git worktrees
     * used during phase execution. Each phase executes in its own worktree
     * at os.tmpdir()/grd-worktree-{milestone}-{phase}.
     *
     * Dependencies: utils.js (one-directional, no circular deps)
     */
    const fs = require('fs');
    const os = require('os');
    const path = require('path');
    const { execGit, output, error, loadConfig, getMilestoneInfo, generateSlugInternal } = require('./utils');
    ```

    **cmdWorktreeCreate(cwd, options, raw):**
    - `options`: `{ phase, milestone, slug }`
    - Compute worktree path: `path.join(os.tmpdir(), \`grd-worktree-\${milestone}-\${phase}\`)`
    - Compute branch: use config `phase_branch_template` pattern: `grd/{milestone}/{phase}-{slug}`
    - If worktree already exists: output error JSON `{ error: "Worktree already exists", path }`
    - Call `execGit(cwd, ['worktree', 'add', '-b', branch, worktreePath])`
    - If git fails: output error JSON with stderr
    - On success: output `{ path, branch, phase, milestone, created: new Date().toISOString() }`

    **cmdWorktreeRemove(cwd, options, raw):**
    - `options`: `{ phase, path: worktreePath, milestone }`
    - Resolve path from phase or direct path
    - If path does not exist on disk: output `{ removed: true, path, already_gone: true }` (graceful)
    - Call `execGit(cwd, ['worktree', 'remove', worktreePath, '--force'], { allowBlocked: true })` (force handles dirty GRD-managed temp worktrees; `--force` is in GIT_BLOCKED_FLAGS so `allowBlocked: true` is required)
    - Call `execGit(cwd, ['worktree', 'prune'])` to clean git state
    - Clean up the temp directory if it still exists: `fs.rmSync(worktreePath, { recursive: true, force: true })`
    - Output `{ removed: true, path }`

    **cmdWorktreeList(cwd, raw):**
    - Call `execGit(cwd, ['worktree', 'list', '--porcelain'])`
    - Parse porcelain output (blocks separated by blank lines, each has `worktree`, `HEAD`, `branch` lines)
    - Filter to only GRD worktrees (path starts with `os.tmpdir()` and contains `grd-worktree-`)
    - For each, extract phase and milestone from the directory name pattern
    - Check if locked: `execGit(cwd, ['worktree', 'list'])` and check for `(locked)` flag
    - Output `{ worktrees: [...], count }` where each entry has `{ path, branch, phase, milestone, locked }`

    **cmdWorktreeRemoveStale(cwd, raw):**
    - Get list of GRD worktrees via internal list logic
    - For each worktree: check if path exists on disk AND if the git worktree is valid
    - A worktree is stale if: path does not exist on disk OR path is empty directory
    - Do NOT remove locked worktrees
    - Remove stale ones via `execGit(cwd, ['worktree', 'remove', path, '--force'], { allowBlocked: true })` + prune
    - Output `{ removed: [...pathsRemoved], count }`

    **Important implementation notes:**
    - Use `execGit` from utils.js (NOT child_process directly) for git commands — this respects security whitelist
    - Note: `worktree` is NOT in GIT_BLOCKED_COMMANDS, so it will work without `allowBlocked` for the subcommand
    - Note: `--force` IS in GIT_BLOCKED_FLAGS — pass `{ allowBlocked: true }` to any execGit call using `--force` (only cmdWorktreeRemove and cmdWorktreeRemoveStale need this)
    - All functions use `output(result, raw)` for JSON output following existing conventions
    - All functions use `error(msg)` for fatal errors following existing conventions
    - JSDoc all exported functions with @param and @returns
    - module.exports at bottom following lib/phase.js pattern
