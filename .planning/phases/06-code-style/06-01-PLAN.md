  <files>package.json, .eslintrc.json, .prettierrc, .prettierignore</files>
  <action>
    1. Install ESLint and Prettier as devDependencies:
       ```
       npm install --save-dev eslint prettier
       ```
       Use latest stable versions. Do NOT install eslint-config-* or eslint-plugin-* packages — start with built-in eslint:recommended only. This minimizes risk of rule conflicts (DEFER-06-01 concern).

    2. Create `.prettierrc` matching the existing code style observed in lib/*.js and .editorconfig:
       ```json
       {
         "singleQuote": true,
         "semi": true,
         "trailingComma": "es5",
         "tabWidth": 2,
         "printWidth": 100,
         "endOfLine": "lf"
       }
       ```
       - singleQuote: true (existing code uses single quotes consistently)
       - semi: true (existing code uses semicolons consistently)
       - trailingComma: "es5" (safe default for Node 18+)
       - tabWidth: 2 (matches .editorconfig indent_size)
       - printWidth: 100 (generous width to avoid unnecessary wrapping in CLI tool code)
       - endOfLine: "lf" (matches .editorconfig end_of_line)

    3. Create `.eslintrc.json` with minimal, safe configuration:
       ```json
       {
         "env": {
           "node": true,
           "es2022": true,
           "jest": true
         },
         "extends": ["eslint:recommended"],
         "parserOptions": {
           "ecmaVersion": 2022
         },
         "rules": {
           "no-unused-vars": ["error", { "argsIgnorePattern": "^_", "caughtErrors": "none" }],
           "no-constant-condition": ["error", { "checkLoops": false }],
           "no-empty": ["error", { "allowEmpty": true }]
         }
       }
       ```
       Key rule overrides:
       - no-unused-vars: ignore args prefixed with _ and caught errors (common in CLI code with try/catch patterns)
       - no-constant-condition: allow while(true) loops (used in CLI dispatch patterns)
       - no-empty: allow empty catch blocks (common in CLI error handling)

    4. Create `.prettierignore` to exclude non-source files:
       ```
       coverage/
       node_modules/
       tests/fixtures/
       tests/golden/
       .planning/
       *.md
       ```
       Exclude test fixtures and golden files (their format is intentional), .planning/ docs, and markdown files (Prettier markdown reformatting can break GRD templates).

    5. Replace placeholder scripts in package.json:
       - `"lint"`: `"eslint bin/ lib/"` (lint source files only, not tests)
       - `"lint:fix"`: `"eslint --fix bin/ lib/"` (auto-fix mode)
       - `"format"`: `"prettier --write bin/ lib/ tests/ jest.config.js"` (format all JS files)
       - `"format:check"`: `"prettier --check bin/ lib/ tests/ jest.config.js"` (CI check mode)

       Keep existing test scripts unchanged.
  </action>
