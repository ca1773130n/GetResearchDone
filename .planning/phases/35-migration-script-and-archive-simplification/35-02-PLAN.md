<objective>
Simplify `cmdMilestoneComplete` archive logic and add a metadata marker for completed milestones.

Purpose: Once directories live under `.planning/milestones/{version}/`, milestone completion should NOT re-copy phase directories to a separate `{version}-phases/` archive. Instead, it should recognize that phases are already in place and simply mark the milestone as archived. This eliminates redundant file operations and simplifies the archive model. A metadata marker (`archived.json`) makes completed milestones distinguishable from active ones.

Output: Simplified `cmdMilestoneComplete` in `lib/phase.js` with archive marker, fully tested.
</objective>

    Modify cmdMilestoneComplete in lib/phase.js with two changes:

    **Change 1 — Conditional phase archive (REQ-59):**

    In the "Archive phase directories" section (around line 817-835), add a check BEFORE the copy+delete logic:

    ```
    // Check if phases are already under the milestone directory (new-style layout)
    const milestonePhaseDir = path.join(cwd, '.planning', 'milestones', version, 'phases');
    const phasesAlreadyInPlace = fs.existsSync(milestonePhaseDir) &&
      fs.readdirSync(milestonePhaseDir, { withFileTypes: true }).some(e => e.isDirectory());
    ```

    If phasesAlreadyInPlace is true:
    - Skip the copy+delete loop entirely
    - Set archivedPhaseCount = number of phase dirs already under milestonePhaseDir
    - Gather stats (phaseCount, totalPlans, totalTasks, accomplishments) from milestonePhaseDir instead of phasesDir

    If phasesAlreadyInPlace is false:
    - Keep existing copy+delete behavior (backward compat for old-style layout)

    Also update the stat-gathering loop (around line 787-815) to read from milestonePhaseDir when phasesAlreadyInPlace is true, since the phases won't be under the old phasesDir.

    **Change 2 — Archive metadata marker (REQ-60):**

    After all archiving is done (both phase copy and roadmap/requirements archive), write an `archived.json` file inside the milestone's archive directory:

    ```javascript
    // Ensure the milestone version directory exists (may not exist in old-style layout)
    fs.mkdirSync(path.join(cwd, '.planning', 'milestones', version), { recursive: true });
    const markerPath = path.join(cwd, '.planning', 'milestones', version, 'archived.json');
    const marker = {
      version,
      name: milestoneName,
      archived_date: today,
      phases: phaseCount,
      plans: totalPlans,
      tasks: totalTasks,
      accomplishments,
    };
    fs.writeFileSync(markerPath, JSON.stringify(marker, null, 2) + '\n', 'utf-8');
    ```

    Update the result object to include:
    - `phases_already_in_place: phasesAlreadyInPlace` (boolean)
    - `archived.marker: true` (the marker was written)
