    **2. Implement generateCleanupPlan(cwd, phaseNum, qualityReport)**
    - Parameters:
      - cwd: project working directory
      - phaseNum: current phase number (string)
      - qualityReport: the quality report object from runQualityAnalysis
    - Logic:
      - Read config via getCleanupConfig(cwd) to get cleanup_threshold (default 5)
      - If qualityReport.summary.total_issues <= cleanup_threshold, return null (no plan needed)
      - Find the next plan number by scanning existing PLAN.md files in the phase directory:
        - List files matching `{phaseNum}-*-PLAN.md` in the phase directory
        - Extract plan numbers, find max, add 1
        - If no plans exist, start at 01
      - Find the phase directory: scan .planning/phases/ for directory starting with `{phaseNum}-`
      - Build PLAN.md content with standard YAML frontmatter:
        ```yaml
        ---
        phase: {phase-slug}
        plan: {next-plan-number}
        type: execute
        wave: 1
        depends_on: []
        files_modified: [list of unique files from quality issues]
        autonomous: true
        verification_level: sanity
        cleanup_generated: true
        must_haves:
          truths:
            - "All auto-detected quality issues from phase {phaseNum} completion are resolved"
          artifacts: []
          key_links: []
        ---
        ```
      - Build task content from quality report details:
        - If complexity violations exist: task to refactor high-complexity functions
        - If dead exports exist: task to remove or document unused exports
        - If file size violations exist: task to split oversized files
        - If doc_drift exists: task to update stale docs (CHANGELOG, README links, JSDoc)
        - Combine related issues into 1-2 tasks (not one task per issue)
      - Write to `.planning/phases/{phase-dir}/{phaseNum}-{nextPlan}-PLAN.md`
      - Return { path: planPath, plan_number: nextPlanNumber, issues_addressed: total_issues }

    - Test: Plan number is sequential after existing plans in the phase directory
    - Test: files_modified in frontmatter reflects files from quality issues
    - Create test fixtures with phase directories containing existing PLAN.md files to verify numbering
