    **Step 2: Implement `cmdLongTermRoadmap(cwd, subcommand, args, raw)` in lib/commands.js**

    Pattern follows the existing `cmdTracker` multi-subcommand style. The function receives:
    - `cwd` - project directory
    - `subcommand` - one of: 'parse', 'validate', 'display', 'mode', 'generate'
    - `args` - remaining CLI args (for file paths, flags)
    - `raw` - boolean for raw vs JSON output

    **Subcommand: `parse`**
    ```
    node grd-tools.js long-term-roadmap parse [file-path]
    ```
    1. `file-path` defaults to `.planning/LONG-TERM-ROADMAP.md` if not provided
    2. Read file content with `safeReadFile`
    3. If file not found: `output({ error: 'LONG-TERM-ROADMAP.md not found', exists: false }, raw, '')`
    4. Call `parseLongTermRoadmap(content)`
    5. Output parsed result as JSON (or raw: milestone count summary)

    **Subcommand: `validate`**
    ```
    node grd-tools.js long-term-roadmap validate [file-path]
    ```
    1. Read and parse the file (same as parse)
    2. Call `validateLongTermRoadmap(parsed)`
    3. Output validation result: `{ valid, errors, warnings }`
    4. Raw mode: output 'valid' or 'invalid: {errors}'

    **Subcommand: `display`**
    ```
    node grd-tools.js long-term-roadmap display
    ```
    1. Read `.planning/LONG-TERM-ROADMAP.md`
    2. If not found: output error
    3. Parse content
    4. Call `formatLongTermRoadmap(parsed)`
    5. JSON mode: `{ formatted: <text>, milestone_count: N, mode: 'hierarchical' }`
    6. Raw mode: output the formatted text directly

    **Subcommand: `mode`**
    ```
    node grd-tools.js long-term-roadmap mode
    ```
    1. Call `getPlanningMode(cwd)`
    2. Output: `{ mode: 'hierarchical'|'progressive', long_term_roadmap_exists: bool }`
    3. Raw mode: output just the mode string

    **Subcommand: `generate`**
    ```
    node grd-tools.js long-term-roadmap generate --project "Name" --horizon "6 months" --milestones <JSON>
    ```
    1. Parse `--project` flag (project name)
    2. Parse `--horizon` flag (planning horizon, default "6 months")
    3. Parse `--milestones` flag (JSON string of milestones array)
    4. If `--milestones` not provided, output error: 'milestones JSON required'
    5. Call `generateLongTermRoadmap(milestones, projectName, horizon)`
    6. Output: `{ content: <generated markdown>, path: '.planning/LONG-TERM-ROADMAP.md' }`
    7. Raw mode: output the generated markdown content

    **Invalid subcommand handling:**
    If subcommand is not recognized: `error('Unknown subcommand: ' + subcommand + '. Valid: parse, validate, display, mode, generate')`

    Export `cmdLongTermRoadmap` from module.exports.
