<objective>
Create the core markdown splitting module with TDD methodology.

Purpose: Build a self-contained module (`lib/markdown-split.js`) that can detect large markdown files, split them into numbered partials at heading boundaries, rewrite the original as an index, and reassemble transparently. This is the foundation that all GRD readers will use in Plan 02.

Output: `lib/markdown-split.js` with 7 exported functions, comprehensive test suite, coverage threshold configured.

Design decisions:
- Token estimation uses ~4 chars/token heuristic (simple, no external deps, good enough for threshold detection)
- Split boundaries are at `##` heading breaks (preserves semantic sections)
- Index file format: original filename rewritten with `<!-- GRD-INDEX -->` marker and links to `{basename}-part{N}.md`
- Partial naming: `{BASENAME}-part{N}.md` (e.g., `STATE-part1.md`, `STATE-part2.md`)
- Default threshold: 25,000 tokens (~100,000 characters)
- The module follows the `cwd`-parameter convention and uses only Node built-ins (fs, path)
</objective>

  <name>Task 1: Write failing tests for markdown splitting module (RED phase)</name>
  <files>tests/unit/markdown-split.test.js</files>
  <action>
    Create comprehensive test suite covering all exported functions:

    1. **estimateTokens(content)**
       - Returns ~0 for empty string
       - Returns approximate count for known content (e.g., 400 chars -> ~100 tokens)
       - Handles multi-byte characters gracefully

    2. **findSplitBoundaries(content, maxTokensPerPartial)**
       - Returns empty array for content below threshold
       - Returns boundary indices at `## ` heading positions
       - Prefers `## ` over `### ` boundaries (higher-level headings first)
       - Never splits mid-paragraph
       - Handles content with no headings (falls back to line-based splitting)

    3. **splitMarkdown(filePath, options)**
       - Splits a file exceeding threshold into `{basename}-part1.md`, `{basename}-part2.md`, etc.
       - Rewrites original file as index with `<!-- GRD-INDEX -->` marker and partial links
       - Returns `{ split: true, partials: [...paths], index: filePath }` when split occurs
       - Returns `{ split: false }` for files under threshold
       - Is idempotent: calling on already-split index file returns `{ split: false }` with no changes
       - Preserves exact content across partials (no byte loss)
       - Uses `options.threshold` to override default 25,000 tokens

    4. **isIndexFile(content)**
       - Returns true for content containing `<!-- GRD-INDEX -->` marker
       - Returns false for regular markdown

    5. **reassembleFromIndex(indexPath)**
       - Reads index file, loads all linked partials, concatenates in order
       - Returns the reassembled content string
       - Throws/returns null if a linked partial is missing

    6. **readMarkdownWithPartials(filePath)**
       - For regular markdown files: returns content as-is (same as fs.readFileSync)
       - For index files: transparently reassembles and returns full content
       - Returns null if file doesn't exist (matches safeReadFile convention)

    7. **Round-trip integrity test**
       - Create a large markdown file (>25k tokens), split it, read it back, compare
       - Content must be identical (byte-for-byte after trimming trailing newlines)

    8. **Edge cases**
       - File with only one heading and content below threshold per partial
       - File with frontmatter (--- block) -- frontmatter stays in part1
       - Empty file
       - File with no headings at all (line-based fallback)

    Use `fs.mkdtempSync` for temp directories. Clean up in afterEach.

    **Constants:**
    - `DEFAULT_TOKEN_THRESHOLD = 25000` (~100,000 characters)
    - `CHARS_PER_TOKEN = 4` (approximation)
    - `INDEX_MARKER = '<!-- GRD-INDEX -->'`

    **Functions to implement:**

    1. `estimateTokens(content)` — Return `Math.ceil(content.length / CHARS_PER_TOKEN)`

    2. `findSplitBoundaries(content, maxTokensPerPartial)` — Scan content for `## ` heading positions. Group consecutive content into partials that stay under `maxTokensPerPartial` tokens. Return array of `{ start, end }` character ranges. If no headings found, fall back to splitting at blank-line boundaries. The first partial always starts at index 0.

    3. `splitMarkdown(filePath, options)` — Read file, check if already an index (return `{ split: false }` if so), estimate tokens, return `{ split: false }` if under threshold. Otherwise: compute boundaries, write each partial as `{dir}/{basename}-part{N}.md`, rewrite original file as index with `INDEX_MARKER`, partial links in markdown list format, and a comment noting the original was auto-split. Return `{ split: true, partials: [...], index: filePath }`.

    4. `isIndexFile(content)` — Return `content.includes(INDEX_MARKER)`

    5. `reassembleFromIndex(indexPath)` — Read the index file, extract partial paths from markdown links (pattern: `[...](partial-path)`), resolve relative to index directory, read each in order, concatenate with no separator (partials are self-contained sections). Return concatenated string. Return null if any partial missing.

    6. `readMarkdownWithPartials(filePath)` — Try to read file; if fails return null. If content contains INDEX_MARKER, delegate to `reassembleFromIndex`. Otherwise return content as-is.

    **Index file format:**
    ```markdown
    <!-- GRD-INDEX -->
    <!-- Auto-split by GRD. Do not edit this file directly. -->
    <!-- Edit the partial files listed below instead. -->

    # {Original title or filename}

    - [Part 1]({basename}-part1.md)
    - [Part 2]({basename}-part2.md)
    - [Part 3]({basename}-part3.md)
    ```
