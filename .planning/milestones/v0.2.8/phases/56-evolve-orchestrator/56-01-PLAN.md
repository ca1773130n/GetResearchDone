    1. **SONNET_MODEL constant**: `const SONNET_MODEL = 'sonnet';` -- the model ceiling for all evolve operations.

    2. **buildDiscoverPrompt()**: Returns prompt string for the discover step. Should invoke `/grd:evolve` discovery subcommand or equivalent skill. Example: `"Use the Skill tool to invoke skill \"grd:discover\" with args \"\". Autonomous mode -- analyze the codebase and report improvement opportunities across productivity, quality, usability, consistency, stability, and new features. Output structured JSON."`.

    3. **buildPlanPrompt(items)**: Returns prompt string for planning selected work items. Takes array of work item descriptions. Instructs Claude to create quick execution plans for each item.

    4. **buildExecutePrompt(item)**: Returns prompt string for executing a single work item. Instructs Claude to implement the specific improvement.

    5. **buildReviewPrompt(item)**: Returns prompt string for reviewing an executed work item. Instructs Claude to verify the change, run tests, and report pass/fail.

    6. **writeEvolutionNotes(cwd, iterationData)**: Appends to `.planning/EVOLUTION.md` (or creates if absent). Each iteration entry includes:
       - `## Iteration N` heading (with timestamp)
       - `### Items Attempted` -- list of work items with pass/fail status
       - `### Decisions Made` -- any significant choices during the iteration
       - `### Patterns Discovered` -- recurring observations
       - `### Takeaways` -- lessons for future iterations
       The function takes `{ iteration, items, outcomes, decisions, patterns, takeaways }` and formats as markdown.

    7. **runEvolve(cwd, options)**: The main orchestration loop. Pattern closely follows `runAutopilot()`:
       ```
       async function runEvolve(cwd, options = {}) {
         const { iterations = 1, itemsPerIteration = 5, timeout, maxTurns, dryRun = false } = options;
         // Import spawnClaude from autopilot.js
         // Load iteration state (from Phase 55's loadIterationState)
         // For each iteration:
         //   1. Discover: call discoverWorkItems(cwd) to get fresh items
         //   2. Merge with existing state's remaining items (Phase 55's merge logic)
         //   3. Select: call selectPriorityItems(mergedItems, itemsPerIteration)
         //   4. For each selected item:
         //      a. Plan: spawnClaude(cwd, buildPlanPrompt([item]), { model: SONNET_MODEL, timeout, maxTurns })
         //      b. Execute: spawnClaude(cwd, buildExecutePrompt(item), { model: SONNET_MODEL, timeout, maxTurns })
         //      c. Review: spawnClaude(cwd, buildReviewPrompt(item), { model: SONNET_MODEL, timeout, maxTurns })
         //      d. Record outcome (pass/fail based on exit codes)
         //   5. Write evolution notes: writeEvolutionNotes(cwd, { iteration, items, outcomes, ... })
         //   6. Persist state: saveIterationState(cwd, updatedState) with remaining + newly-discovered items
         // Return summary { iterations_completed, items_attempted, items_passed, items_failed, evolution_notes_path }
       ```
       CRITICAL: Every spawnClaude call MUST pass `{ model: SONNET_MODEL }` to enforce the sonnet ceiling. No exceptions.

    8. **cmdInitEvolve(cwd, raw)**: Pre-flight context function (following cmdInitAutopilot pattern). Returns JSON with:
       - `iteration_state`: loaded from disk (or null if first run)
       - `config`: relevant config fields (autonomous_mode, model_profile)
       - `model_ceiling`: SONNET_MODEL value
       - `evolution_notes_exist`: boolean
       - `previous_iterations`: count from state file

    9. **cmdEvolve(cwd, args, raw)**: CLI entry point (following cmdAutopilot pattern). Parses --iterations, --items, --timeout, --max-turns, --dry-run flags from args and calls runEvolve().
