  <name>Task 1: Add evolve CLI commands, cmdInitEvolve, and wire bin/grd-tools.js routing</name>
  <files>lib/evolve.js, bin/grd-tools.js</files>
  <action>
    **Step 1: Add cmd* CLI entry point functions to lib/evolve.js**

    Add new imports at the top of `lib/evolve.js` (needed for cmdInitEvolve):
    - `const { detectBackend, getBackendCapabilities } = require('./backend');`

    Add a new section `// ─── CLI Command Functions ─────────────────────────────────────────────────`

    1. `cmdEvolveDiscover(cwd, args, raw)` — runs discovery and outputs results:
       - Parse `--count N` flag from args (default: DEFAULT_ITEMS_PER_ITERATION)
       - Call `readEvolveState(cwd)` to get previous state (may be null)
       - Call `runDiscovery(cwd, previousState)`
       - Output JSON: `{ selected, remaining, all_discovered_count, merged_count, items_per_iteration }`
       - Raw output: `selected.length` items as count string

    2. `cmdEvolveState(cwd, _args, raw)` — reads and outputs the current evolve state:
       - Call `readEvolveState(cwd)`
       - If null, output `{ exists: false, state: null }`
       - If exists, output `{ exists: true, state: <full state object> }`
       - Raw output: JSON.stringify of state or 'No evolve state found'

    3. `cmdEvolveAdvance(cwd, _args, raw)` — advances to next iteration:
       - Call `readEvolveState(cwd)`
       - If null, output error: 'No evolve state found. Run discover first.'
       - Call `advanceIteration(previousState)`
       - Call `writeEvolveState(cwd, newState)`
       - Output the new state JSON
       - Raw output: `iteration ${newState.iteration}` string

    4. `cmdEvolveReset(cwd, _args, raw)` — deletes the evolve state file (start fresh):
       - Get path via `evolveStatePath(cwd)`
       - If file exists, delete it with `fs.unlinkSync`
       - Output `{ reset: true }`
       - Raw output: 'Evolve state reset'

    5. `cmdInitEvolve(cwd, raw)` — pre-flight context for the evolve workflow (follows the `cmdInitAutopilot` pattern from `lib/autopilot.js`, NOT `lib/context.js`):
       - Call `loadConfig(cwd)` for config
       - Call `detectBackend(cwd)` and `getBackendCapabilities(backend)` from `lib/backend.js`
       - Call `readEvolveState(cwd)` for existing state (may be null)
       - Use `resolveModelForAgent` from `./utils` to resolve planner and executor models
       - Use `getMilestoneInfo` from `./utils` for milestone context
       - Output JSON:
         ```
         {
           backend, capabilities,
           config: { model_profile, autonomous_mode, items_per_iteration },
           evolve_state: { exists, iteration, remaining_count, bugfix_count },
           models: { planner, executor },
           milestone,
         }
         ```
       - Raw output: JSON.stringify of result

    Add all 5 cmd* functions to `module.exports`.

    **Step 2: Wire routing in bin/grd-tools.js**

    At the top of `bin/grd-tools.js`, add the import:
    ```js
    const {
      cmdEvolveDiscover,
      cmdEvolveState,
      cmdEvolveAdvance,
      cmdEvolveReset,
      cmdInitEvolve,
    } = require('../lib/evolve');
    ```

    In the `routeCommand` switch statement, add a new case for `'evolve'` (add it near the `'autopilot'` case since they are related orchestration commands):
    ```js
    case 'evolve': {
      const sub = args[0];
      validateSubcommand(sub, ['discover', 'state', 'advance', 'reset'], 'evolve');
      switch (sub) {
        case 'discover':
          return cmdEvolveDiscover(cwd, args.slice(1), raw);
        case 'state':
          return cmdEvolveState(cwd, args.slice(1), raw);
        case 'advance':
          return cmdEvolveAdvance(cwd, args.slice(1), raw);
        case 'reset':
          return cmdEvolveReset(cwd, args.slice(1), raw);
      }
      break;
    }
    ```

    In the `case 'init':` switch block, add:
    ```js
    case 'evolve':
      return cmdInitEvolve(cwd, raw);
    ```

    Also add `'evolve'` to the `VALID_COMMANDS` array (or equivalent list) if one exists in the top-level command validation.
  </action>
