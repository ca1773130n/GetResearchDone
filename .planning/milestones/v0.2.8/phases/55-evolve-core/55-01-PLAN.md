  <name>Task 1: Create lib/evolve.js with state data structures and disk I/O</name>
  <files>lib/evolve.js</files>
  <action>
    Create `lib/evolve.js` following the established module conventions (JSDoc header, require Node built-ins first then local modules, section separators, module.exports at bottom).

    **Imports needed:**
    - `fs`, `path` from Node built-ins
    - `safeReadFile`, `output`, `error` from `./utils`

    **Constants:**
    - `EVOLVE_STATE_FILENAME = 'EVOLVE-STATE.json'` — state file name
    - `WORK_ITEM_DIMENSIONS` — array of 6 dimensions: `['productivity', 'quality', 'usability', 'consistency', 'stability', 'new-features']`
    - `DEFAULT_ITEMS_PER_ITERATION = 5`

    **Work item data structure** (documented in JSDoc):
    ```
    {
      id: string,           // deterministic: `${dimension}/${slug}`
      dimension: string,    // one of WORK_ITEM_DIMENSIONS
      slug: string,         // kebab-case identifier
      title: string,        // human-readable title
      description: string,  // what needs to change and why
      effort: string,       // 'small' | 'medium' | 'large'
      source: string,       // 'discovery' | 'bugfix' | 'carryover'
      status: string,       // 'pending' | 'selected' | 'completed' | 'failed'
      iteration_added: number, // iteration number when first discovered
    }
    ```

    **Iteration state data structure:**
    ```
    {
      iteration: number,         // 1-based iteration counter
      timestamp: string,         // ISO 8601 timestamp
      milestone: string,         // current milestone version
      items_per_iteration: number,
      selected: WorkItem[],      // items picked for this iteration
      remaining: WorkItem[],     // items not yet attempted
      bugfix: WorkItem[],        // newly-discovered bugfix items
      completed: WorkItem[],     // items completed in this iteration
      failed: WorkItem[],        // items that failed in this iteration
      history: Array<{           // summary of past iterations
        iteration: number,
        timestamp: string,
        selected_count: number,
        completed_count: number,
        failed_count: number,
      }>,
    }
    ```

    **Functions to implement:**

    1. `evolveStatePath(cwd)` — returns the path to the evolve state file. Use `path.join(cwd, '.planning', EVOLVE_STATE_FILENAME)`. This file lives at the project root level of .planning/, not milestone-scoped, because the evolve loop operates across milestones.

    2. `createWorkItem(dimension, slug, title, description, opts = {})` — factory function that creates a work item object with defaults (effort: 'medium', source: 'discovery', status: 'pending', iteration_added: 1). Validates dimension is in WORK_ITEM_DIMENSIONS. Returns the work item object. The `id` field is computed as `${dimension}/${slug}`.

    3. `readEvolveState(cwd)` — reads and parses the evolve state JSON from disk. Returns `null` if file doesn't exist. Uses `safeReadFile`. Wraps JSON.parse in try/catch, returns `null` on parse failure.

    4. `writeEvolveState(cwd, state)` — writes the evolve state JSON to disk with 2-space indentation. Creates parent directory if needed via `fs.mkdirSync(dir, { recursive: true })`.

    5. `createInitialState(milestone, itemsPerIteration)` — creates a fresh state object with iteration=1, empty arrays, empty history, current timestamp.

    6. `mergeWorkItems(existing, discovered)` — merges two arrays of work items, deduplicating by `id` field. Items from `discovered` that share an `id` with `existing` are dropped (existing wins). Returns the merged array. This is the core deduplication logic for REQ-55.

    7. `advanceIteration(previousState)` — creates a new state object for the next iteration:
       - Increments `iteration` counter
       - Moves `remaining` items from previous state (status still 'pending') to new state's `remaining`
       - Moves `bugfix` items from previous state to new state's `remaining` (merged)
       - Appends a history entry summarizing the previous iteration
       - Resets `selected`, `completed`, `failed` to empty arrays
       - Sets new timestamp

    All functions that don't call output/error should be pure (no process.exit). Keep cmd* functions separate for CLI entry points (those will come in Plan 03).
  </action>
