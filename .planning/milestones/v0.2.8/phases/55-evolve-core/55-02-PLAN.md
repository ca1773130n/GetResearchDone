  <action>
    Add the discovery engine and priority selection to `lib/evolve.js`. Add new imports at the top:
    - `const { analyzeRoadmap } = require('./roadmap');`
    - `const { getCleanupConfig } = require('./cleanup');` (for quality signal detection)

    **New section: Discovery Engine**

    Add a section separator: `// ─── Discovery Engine ────────────────────────────────────────────────────`

    1. `analyzeCodebaseForItems(cwd)` — the main discovery function. Returns an array of WorkItem objects. This function performs deterministic analysis (no LLM calls — pure code analysis):

       **Productivity dimension:**
       - Scan `lib/` files for functions exceeding ~80 lines (split candidates)
       - Check for duplicate code patterns: files with very similar exports
       - Look for missing `cmdInit*` functions in context.js that could be added for new workflows

       **Quality dimension:**
       - Read `jest.config.js` and identify any modules below 90% line coverage threshold (these are improvement targets)
       - Check for files in `lib/` that lack corresponding test files in `tests/unit/`
       - Look for TODO/FIXME/HACK comments in lib/ files

       **Usability dimension:**
       - Check `commands/` directory for commands without descriptions in frontmatter
       - Scan for undocumented exported functions (missing JSDoc)
       - Look for commands listed in CLAUDE.md that might have stale descriptions

       **Consistency dimension:**
       - Check for modules that don't follow the `cmd*` prefix convention for CLI handlers
       - Look for files missing the standard JSDoc header pattern
       - Check for inconsistent error handling (direct process.exit vs error() calls)

       **Stability dimension:**
       - Identify files with empty catch blocks that swallow errors silently
       - Check for hardcoded paths (not using lib/paths.js)
       - Look for missing input validation on exported functions

       **New features dimension:**
       - Check if there are `init` workflow types that don't have corresponding MCP tools
       - Look for patterns in existing commands that suggest natural extensions

       Implementation approach: Each dimension gets an internal helper function (e.g., `discoverProductivityItems(cwd)`, `discoverQualityItems(cwd)`, etc.) that returns an array of work items for that dimension. The main function calls all six and concatenates results.

       Keep discovery lightweight — use `fs.readFileSync`, `fs.readdirSync`, and string/regex matching. No subprocess spawning. Each helper should be wrapped in try/catch returning empty array on failure (defensive — don't crash if a file is missing).

       Each discovered item should have a descriptive `title` and `description` that would let an executor understand what to do without additional context.

    2. `scoreWorkItem(item)` — scoring heuristic for priority selection. Returns a numeric score (higher = more important). Scoring factors:
       - `dimension` weight: quality=10, stability=9, consistency=7, productivity=6, usability=5, new-features=3
       - `effort` modifier: small=3, medium=2, large=1 (prefer low-hanging fruit)
       - `source` modifier: bugfix=5, discovery=2, carryover=1 (prefer bugfixes)
       - Final score = dimension_weight + effort_modifier + source_modifier

    3. `selectPriorityItems(items, count)` — selects top N items by score. Steps:
       - Score all items using `scoreWorkItem`
       - Sort descending by score (stable sort — items with equal score keep original order)
       - Take first `count` items, mark their status as 'selected'
       - Return `{ selected: WorkItem[], remaining: WorkItem[] }`

    4. `runDiscovery(cwd, previousState)` — orchestrates full discovery flow:
       - Call `analyzeCodebaseForItems(cwd)` to get fresh items
       - If `previousState` exists, merge fresh items with `previousState.remaining` using `mergeWorkItems`
       - If `previousState` exists, also merge in `previousState.bugfix` items
       - Call `selectPriorityItems` on merged list with `previousState.items_per_iteration` or `DEFAULT_ITEMS_PER_ITERATION`
       - Return `{ selected, remaining, all_discovered_count, merged_count }`

    Add all 4 new functions to `module.exports`.

    **IMPORTANT patterns to follow:**
    - All functions accept `cwd` where filesystem access is needed
    - Internal helpers (discover*Items) are NOT exported — only the 4 main functions above
    - Use `safeReadFile` from utils.js for all file reads
    - All file paths constructed via `path.join()`, never string concatenation
    - Defensive try/catch around each discovery dimension (one failing dimension shouldn't block others)
