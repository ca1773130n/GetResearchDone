  <name>Task 1: Migrate bin/grd-mcp-server.js to bin/grd-mcp-server.ts and create CJS proxy</name>
  <files>bin/grd-mcp-server.ts, bin/grd-mcp-server.js</files>
  <action>
    Create bin/grd-mcp-server.ts by migrating the existing bin/grd-mcp-server.js to TypeScript:

    1. Preserve the shebang line: `#!/usr/bin/env node`
    2. Add 'use strict' directive
    3. Import McpServer with typed require-as cast:
       ```typescript
       const { McpServer } = require('../lib/mcp-server.ts') as typeof import('../lib/mcp-server.ts');
       ```
       Or, if mcp-server.ts exports types separately, import the McpServer type for the class. The key is that `server` has the correct type so that `server.handleMessage(message)` is type-checked.

    4. Type the `server` variable: `const server = new McpServer({ cwd: process.cwd() });`
       (Type is inferred from the constructor return type)

    5. Type the `buffer` variable: `let buffer: string = '';`

    6. Type the stdin data handler: the `chunk` parameter should be typed as `string` (since `setEncoding('utf-8')` is called)

    7. Type the `message` variable in the JSON parse block as the JsonRpcMessage type from mcp-server.ts. If that type is not exported, define a local interface:
       ```typescript
       interface JsonRpcMessage {
         jsonrpc?: string;
         method?: string;
         id?: number | string | null;
         params?: Record<string, unknown>;
       }
       ```

    8. Type the `response` variable: the return type of `server.handleMessage()` which can be `JsonRpcResponse | Promise<JsonRpcResponse> | null`

    9. The `Promise.resolve(response).then(...)` pattern works as-is since it handles both sync and async responses

    10. Type the error response objects inline: `{ jsonrpc: '2.0' as const, id: null, error: { code: -32700, message: 'Parse error' } }`

    11. In catch blocks, use `(_e: unknown)` pattern since the catch blocks intentionally ignore the error

    Then convert bin/grd-mcp-server.js to a CJS proxy:
    ```javascript
    #!/usr/bin/env node
    /**
     * GRD MCP Server -- CommonJS re-export proxy
     *
     * Canonical implementation lives in grd-mcp-server.ts; this proxy
     * re-exports it for runtime CJS resolution.
     */
    'use strict';
    require('./grd-mcp-server.ts');
    ```
  </action>
