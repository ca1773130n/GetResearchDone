<task type="auto">
  <name>Task 2: Implement runMultiMilestoneAutopilot orchestration loop</name>
  <files>lib/autopilot.ts</files>
  <action>
    Add the main `runMultiMilestoneAutopilot` function in lib/autopilot.ts. This is an async function that:

    1. **Initialization:**
       - Accept `cwd: string` and `options: MultiMilestoneOptions` parameters
       - Set defaults: maxMilestones = 10 (safety cap to prevent infinite loops)
       - Create the autopilot log file and log function (reuse existing pattern)
       - Initialize result tracking: milestones_attempted, milestones_completed, milestone_results array

    2. **Main loop** (iterate up to maxMilestones times):

       a. **Get current milestone info:**
          ```typescript
          const milestoneInfo = getMilestoneInfo(cwd);
          ```

       b. **Check if current milestone has incomplete phases:**
          - Call `resolvePhaseRange(cwd, null, null)` to get all phases
          - If there are incomplete phases, run `runAutopilot(cwd, { resume: options.resume, ... })` on them
          - Record the per-milestone result

       c. **Check milestone completion after autopilot run:**
          - If `isMilestoneComplete(cwd)` returns true:
            - Log milestone completion
            - If NOT dryRun, spawn `buildMilestoneCompletePrompt(milestoneInfo.version)` via spawnClaude to complete the milestone
            - Increment milestones_completed

       d. **Resolve next milestone:**
          - Call `resolveNextMilestone(cwd)`
          - If null, log "no next milestone found" and break
          - If dryRun, log what would happen and continue counting
          - Otherwise, spawn `buildNewMilestonePrompt()` via spawnClaude to create the next milestone
          - If spawn fails, log error and break

       e. **Loop back** to process the newly created milestone

    3. **Return MultiMilestoneResult** with aggregated stats.

    Key design decisions:
    - The milestone complete step uses the deterministic `grd-tools.js milestone complete` command (no LLM needed)
    - The new-milestone step uses `claude -p` with the /grd:new-milestone skill (LLM needed for research, requirements, roadmap creation)
    - Each milestone iteration gets a fresh check of the roadmap state (no stale caches)
    - The `maxMilestones` safety cap prevents runaway loops
    - All spawns use the same timeout/maxTurns/model settings from options

    4. **Export** the new function in module.exports.
