  <action>
    1. Install TypeScript ESLint packages:
       npm install --save-dev @typescript-eslint/parser @typescript-eslint/eslint-plugin typescript-eslint

       Note: The `typescript-eslint` package is the modern unified package that provides
       both parser and plugin for ESLint flat config. It exports `tseslint.configs.recommended`
       and `tseslint.configs.strictTypeChecked` presets.

    2. Rewrite eslint.config.js to support BOTH JavaScript and TypeScript files.
       The config must use ESLint flat config format (array of config objects).

       Architecture:
       - Keep the existing @eslint/js recommended config for JS files
       - Add typescript-eslint configs for .ts files
       - Use a file-pattern-based approach: .js files get JS rules, .ts files get TS rules
       - Preserve ALL existing rules exactly (no-unused-vars with argsIgnorePattern: "^_",
         no-constant-condition with checkLoops: false, no-empty with allowEmptyCatch: true)

       Implementation approach using the typescript-eslint v8+ flat config API:

       ```javascript
       'use strict';

       const js = require('@eslint/js');
       const globals = require('globals');
       const tseslint = require('typescript-eslint');

       module.exports = [
         // Base JS recommended rules (applies to all files)
         js.configs.recommended,

         // TypeScript recommended rules (applies to .ts files only)
         ...tseslint.configs.recommended.map(config => ({
           ...config,
           files: ['**/*.ts'],
         })),

         // Shared settings for all JS and TS files
         {
           languageOptions: {
             ecmaVersion: 2022,
             globals: {
               ...globals.node,
               ...globals.jest,
             },
           },
           rules: {
             'no-unused-vars': [
               'error',
               { argsIgnorePattern: '^_', caughtErrors: 'none', ignoreRestSiblings: true },
             ],
             'no-constant-condition': ['error', { checkLoops: false }],
             'no-empty': ['error', { allowEmptyCatch: true }],
           },
         },

         // TypeScript-specific overrides
         {
           files: ['**/*.ts'],
           languageOptions: {
             parser: tseslint.parser,
             parserOptions: {
               projectService: true,
               tsconfigRootDir: __dirname,
             },
           },
           rules: {
             // Disable base no-unused-vars in favor of TS version for .ts files
             'no-unused-vars': 'off',
             '@typescript-eslint/no-unused-vars': [
               'error',
               { argsIgnorePattern: '^_', caughtErrors: 'none', ignoreRestSiblings: true },
             ],
             // Allow require() for CommonJS compatibility during migration
             '@typescript-eslint/no-require-imports': 'off',
           },
         },
       ];
       ```

       CRITICAL: The `no-unused-vars` rule MUST be disabled for .ts files and replaced
       with `@typescript-eslint/no-unused-vars` using the EXACT same options (argsIgnorePattern: "^_",
       caughtErrors: "none", ignoreRestSiblings: true). Otherwise TypeScript-specific syntax
       like type imports will trigger false positives.

       CRITICAL: `@typescript-eslint/no-require-imports` must be set to 'off' because the project
       uses CommonJS (require/module.exports), not ESM. The default recommended config enables
       this rule which would flag every require() call as an error.

    3. Lint scripts in package.json do NOT need changes.
       Keep them as-is: "lint": "eslint bin/ lib/" and "lint:fix": "eslint --fix bin/ lib/"

       The --ext flag was removed in ESLint v9 flat config (and is not supported in v10).
       ESLint flat config discovers .ts files automatically via the files: ['**/*.ts'] pattern
       in the config objects above. No script changes needed.
