<task type="auto">
  <name>Task 1: Install TypeScript and create tsconfig.json with strict mode</name>
  <files>package.json, tsconfig.json, tsconfig.build.json</files>
  <action>
    1. Install TypeScript as a dev dependency:
       npm install --save-dev typescript

    2. Create tsconfig.json at project root with these settings:
       - compilerOptions:
         - strict: true (all strict checks enabled)
         - target: "ES2022" (matches current ecmaVersion 2022 in eslint config)
         - module: "commonjs" (preserve existing CommonJS format)
         - moduleResolution: "node"
         - esModuleInterop: true
         - resolveJsonModule: true
         - declaration: true
         - declarationMap: true
         - sourceMap: true
         - skipLibCheck: true
         - forceConsistentCasingInFileNames: true
         - allowJs: true (enables incremental migration — .js and .ts coexist)
         - checkJs: false (don't type-check existing JS files yet)
         - noEmit: true (base config is for type-checking only — build output handled by tsconfig.build.json)
         - rootDir: "."
       - include: ["lib/**/*.ts", "bin/**/*.ts"]
       - exclude: ["node_modules", "dist", "coverage", "tests", "testbed"]

    3. Create tsconfig.build.json that extends tsconfig.json:
       - extends: "./tsconfig.json"
       - compilerOptions:
         - noEmit: false (override base — build config DOES produce output)
         - declaration: true
         - declarationMap: true
         - sourceMap: true
         - outDir: "./dist"
       - include: ["lib/**/*.ts", "bin/**/*.ts"]
       - exclude: ["node_modules", "dist", "coverage", "tests", "testbed", "**/*.test.ts"]

    IMPORTANT: allowJs: true is critical so that existing .js files in lib/ and bin/ continue to work
    without requiring immediate migration. checkJs: false avoids breaking existing code with type errors.
  </action>
  <verify>
    npx tsc --version (confirms TypeScript installed)
    cat tsconfig.json | node -e "const fs=require('fs'); const c=JSON.parse(fs.readFileSync('tsconfig.json','utf8')); console.log('strict:', c.compilerOptions.strict, 'module:', c.compilerOptions.module)" (Level 1: Sanity)
  </verify>
  <done>
    TypeScript installed, tsconfig.json exists with strict:true and module:"commonjs",
    tsconfig.build.json extends it with dist/ output configuration
  </done>

       Example structure:
       ```typescript
       /**
        * GRD Sample TypeScript Module — Validates TypeScript toolchain
        *
        * This module exists to prove the TypeScript compilation pipeline works.
        * It will be removed or replaced once real migrations begin in Phase 59.
        */

       interface PhaseInfo {
         number: number;
         name: string;
         status: 'not_started' | 'in_progress' | 'completed' | 'skipped';
       }

       function createPhaseInfo(number: number, name: string): PhaseInfo {
         if (number < 0) {
           throw new Error(`Phase number must be non-negative, got ${number}`);
         }
         return { number, name, status: 'not_started' };
       }

       function formatPhaseLabel(phase: PhaseInfo): string {
         return `Phase ${String(phase.number).padStart(2, '0')}: ${phase.name} [${phase.status}]`;
       }

       module.exports = { createPhaseInfo, formatPhaseLabel };
       ```
