       CRITICAL: The transform config ONLY applies to .ts files. When a `transform` object
       is provided with ONLY a `.ts` entry (and no `.js` entry), Jest does NOT apply any
       transform to .js files — they are loaded natively via Node's CommonJS require().
       This is the correct behavior for this project's existing CommonJS .js test files.
       The 2,184 existing JS tests continue to work identically because:
       1. No .js transform key exists → Jest skips transformation for .js files
       2. Node's native require() handles CommonJS .js files directly
       3. This is the same behavior as having no transform config at all for .js files

       Add a comment in the jest.config.js template above the transform key:
       ```javascript
       // Only .ts files are transformed (via ts-jest). .js files have no transform
       // entry, so Jest loads them natively via Node's CommonJS require() — identical
       // to the pre-TypeScript behavior. This preserves all existing JS test behavior.
       transform: {
         '^.+\\.ts$': 'ts-jest',
       },
       ```

       CRITICAL: Exclude .d.ts files from coverage collection (they are type declarations,
       not executable code). The pattern '!lib/**/*.d.ts' handles this.

       CRITICAL: Preserve every single existing coverage threshold. Copy them exactly
       from the current jest.config.js — do not round, adjust, or omit any.

       ```typescript
        * Validates TypeScript test compilation via ts-jest.

       const { createPhaseInfo, formatPhaseLabel } = require('../../lib/sample');

       describe('sample TypeScript module', () => {
         describe('createPhaseInfo', () => {
           it('should create phase info with not_started status', () => {
             const phase = createPhaseInfo(1, 'foundation');
             expect(phase).toEqual({
               number: 1,
               name: 'foundation',
               status: 'not_started',
             });
           });

           it('should accept zero as a valid phase number', () => {
             const phase = createPhaseInfo(0, 'init');
             expect(phase.number).toBe(0);
           });

           it('should throw for negative phase numbers', () => {
             expect(() => createPhaseInfo(-1, 'bad')).toThrow('non-negative');
           });
         });

         describe('formatPhaseLabel', () => {
           it('should format single-digit phase with zero padding', () => {
             const phase = createPhaseInfo(3, 'toolchain');
             expect(formatPhaseLabel(phase)).toBe('Phase 03: toolchain [not_started]');
           });

           it('should format double-digit phase without extra padding', () => {
             const phase = createPhaseInfo(58, 'typescript');
             expect(formatPhaseLabel(phase)).toBe('Phase 58: typescript [not_started]');
           });
         });
       });
       ```
