  <name>Task 1: Update CI pipeline with TypeScript build gates and Node 18 support</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Update `.github/workflows/ci.yml` with these changes:

    **1. Add type-check step to the lint job:**
    After the existing "Lint" step, add:
    ```yaml
    - name: Type check
      run: npm run build:check
    ```

    **2. Add build step to the test job (before Test):**
    ```yaml
    - name: Build
      run: npm run build

    - name: Verify dist/ build works
      run: |
        node dist/bin/grd-tools.js current-timestamp --raw
        echo "dist/ CLI: OK"
    ```

    **3. Expand Node version matrix to include 18:**
    Change:
    ```yaml
    matrix:
      node-version: [20, 22]
    ```
    To:
    ```yaml
    matrix:
      node-version: [18, 20, 22]
    ```

    **4. Update the validate job to use dist/ build:**
    In the "Pack and install validation" step, after `npm install`, the verification commands should test the compiled dist/ output. The key insight: when the package is installed from tarball, the CJS proxies in bin/ require .ts files. The consumer must either have ts-jest or use the dist/ build. For npm distribution, package.json should eventually point `bin` entries to `dist/bin/`, but for now the validate job should verify both:
    - Source mode: `node node_modules/grd-tools/bin/grd-tools.js current-timestamp --raw` (works if Node supports .ts via proxies)
    - Dist mode: `node node_modules/grd-tools/dist/bin/grd-tools.js current-timestamp --raw` (always works)

    Update the validate step to test dist/ mode as the primary path:
    ```yaml
    - name: Pack and install validation
      run: |
        # Build dist/ first
        npm run build

        # Pack the tarball
        npm pack
        TARBALL=$(ls grd-tools-*.tgz)
        echo "Packed: $TARBALL"

        # Create a temp consumer project
        INSTALL_DIR=$(mktemp -d)
        cd "$INSTALL_DIR"
        npm init -y --silent
        npm install "$GITHUB_WORKSPACE/$TARBALL"

        # Verify dist/ build is functional in consumer context
        node node_modules/grd-tools/dist/bin/grd-tools.js current-timestamp --raw
        echo "grd-tools dist/ CLI: OK"

        # Verify MCP server starts and responds to initialize handshake
        MCP_RESPONSE=$(echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"ci-test","version":"1.0.0"}}}' | timeout 5 node node_modules/grd-tools/dist/bin/grd-mcp-server.js)
        echo "$MCP_RESPONSE" | grep -q '"protocolVersion"'
        echo "grd-mcp-server dist/: OK"

        # Verify plugin.json exists at expected path
        test -f node_modules/grd-tools/.claude-plugin/plugin.json
        echo "plugin.json: OK"

        # Clean up
        cd "$GITHUB_WORKSPACE"
        rm -rf "$INSTALL_DIR"
        rm -f "$TARBALL"
    ```

    **NOTE:** The package.json `files` field update (adding `dist/`) is handled in Task 2 Part C below to avoid duplication.

    **IMPORTANT:** The lint job node-version should stay at 22 (latest for linting). Only the test matrix expands to 18/20/22.
  </action>

    **Part A: Fix npm-pack.test.ts**

    The npm-pack test currently fails because:
    1. The packed tarball includes CJS proxy .js files that `require('./foo.ts')`
    2. In the consumer's node_modules, Node tries to load .ts files but fails without ts-jest or --experimental-strip-types
    3. The postinstall.js script runs during `npm install` and fails for the same reason

    Fix approach:
    - Update the test to build dist/ first (`npm run build` in beforeAll)
    - Update the test to validate dist/ paths instead of source paths
    - The consumer should use `dist/bin/grd-tools.js` instead of `bin/grd-tools.js`
    - For the postinstall script: it runs automatically during npm install. Since it requires postinstall.ts, the test needs to account for this. Options:
      a. Add `--ignore-scripts` flag to the npm install in the test
      b. Or make postinstall.js try dist/ fallback
    - Use `npm install --ignore-scripts` in the test and verify dist/ paths directly

    Update the test assertions to:
    - Check dist/bin/grd-tools.js exists in installed package
    - Check dist/bin/grd-mcp-server.js exists in installed package
    - Run `node node_modules/grd-tools/dist/bin/grd-tools.js current-timestamp --raw` successfully
    - Check plugin.json exists
