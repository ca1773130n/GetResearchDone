<objective>
Fix all 65 `.ts` require() paths in TypeScript source files to use extensionless imports, making the `dist/` compiled build functional as a drop-in replacement.

Purpose: The TypeScript compiler preserves require() paths verbatim in compiled output. Currently, source files use `require('../lib/utils.ts')` which works at runtime via Node's `.ts` support (ts-jest, Node 24 experimental), but the compiled `dist/` JS files end up with `require('../lib/utils.ts')` instead of `require('../lib/utils')`, causing MODULE_NOT_FOUND errors. This is the single blocker preventing dist/ from being a functional build.

Output: All 18 TS source files with extensionless require() paths. A verified dist/ build where `node dist/bin/grd-tools.js` runs successfully.
</objective>

  <action>
    In every .ts source file under bin/ and lib/, change all require() paths that end in `.ts` to extensionless paths:

    Pattern: `require('./foo.ts')` -> `require('./foo')`
    Pattern: `require('../lib/utils.ts')` -> `require('../lib/utils')`

    The `as { ... }` type cast after the require stays unchanged.

    **Why this works:** TypeScript's `moduleResolution: "node"` resolves extensionless `require('./foo')` to `./foo.ts` during type-checking (because `allowJs: true` and .ts files exist). In the compiled dist/ output, the same extensionless `require('./foo')` resolves to `./foo.js` (because tsc compiles .ts -> .js). This is the standard TypeScript CJS pattern.

    **Affected files (18 files, 65 require paths):**

    bin/ files (4):
    - `grd-tools.ts`: ~16 require paths (../lib/utils.ts, ../lib/frontmatter.ts, etc.)
    - `grd-mcp-server.ts`: 1 require path (../lib/mcp-server.ts)
    - `grd-manifest.ts`: check for any .ts requires
    - `postinstall.ts`: check for any .ts requires

    lib/commands/ files (5):
