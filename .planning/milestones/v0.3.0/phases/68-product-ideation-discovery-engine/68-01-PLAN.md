    1. `gatherProductContext(cwd: string): ProductIdeationContext`
       - Read `.planning/PROJECT.md` using `safeReadFile`. Extract the first 3000 chars (vision section). Store in `projectVision`.
       - Read `.planning/LONG-TERM-ROADMAP.md` using `safeReadFile`. Extract the first 2000 chars. Store in `longTermGoals`.

       - List files in `commands/` directory. Filter for `.md` files. Map to command names (strip `.md` extension). Store in `existingCommands`.
       - List files in `agents/` directory. Filter for `.md` files. Map to agent names (strip `.md` extension). Store in `existingAgents`.
       - Read `.planning/ROADMAP.md` using `safeReadFile`. Extract the last ~2000 chars (recent phases section). Store in `recentPhases`.
       - Return the `ProductIdeationContext` object.
       - Wrap each file read in try/catch -- if any file is missing, use null or empty array as fallback.

    2. `buildProductIdeationPrompt(context: ProductIdeationContext): string`
       - Build a prompt that is fundamentally different from `buildDiscoveryPrompt` in discovery.ts.
       - The prompt MUST:
         a. Set the role: "You are a product manager for a developer tools company."
         b. Provide the project vision from PROJECT.md
         c. Provide the long-term roadmap goals
         d. List existing commands and agents as "what already exists"
         e. Provide recent phases as "what was just built" context
         f. Include product quality gaps if available
         g. Ask Claude to generate creative feature ideas across these categories:
            - New user-facing commands or workflows
            - New integrations with external tools/services
            - UX improvements to existing workflows
            - Developer experience enhancements
            - New analysis or insight capabilities
            - Automation or productivity features
         h. Specify the output format: JSON array with `{"dimension":"product-ideation","slug":"<kebab-id>","title":"<short>","description":"<what+why+user-value>","effort":"small|medium|large"}`
         i. Specify rules:
            - Think about USER VALUE, not code hygiene
            - Each idea should solve a real user problem or unlock a new capability
            - Be creative but realistic -- ideas should be implementable
            - Include the "why" -- what user pain point does this address?
            - 15-40 items
            - ONLY the JSON array, no other text
       - Return the complete prompt string.

    3. `parseProductIdeationOutput(raw: string): WorkItem[]`
       - Same parsing logic as `parseDiscoveryOutput` in discovery.ts (strip markdown fences, JSON.parse, validate fields).
       - IMPORTANT: Only accept items where `dimension === 'product-ideation'`. Reject items with other dimensions.
       - Use `createWorkItem` to construct validated WorkItem objects.
       - Accept effort values of 'small', 'medium', 'large' (default to 'medium' if invalid).
       - Return empty array if parsing fails (no crash).

    4. `async function discoverProductIdeationItems(cwd: string): Promise<WorkItem[]>`
       - Call `gatherProductContext(cwd)` to collect context.
       - If `context.projectVision` is null (no PROJECT.md), return empty array with a stderr warning: `[evolve] Product ideation skipped: no PROJECT.md found`.
       - Call `buildProductIdeationPrompt(context)` to build the prompt.
       - Call `spawnClaudeAsync(cwd, prompt, { captureOutput: true, model: SONNET_MODEL, maxTurns: 15, timeout: 120_000, outputFormat: 'text' })`.
       - If subprocess fails (exitCode !== 0, timedOut, or stdout starts with 'Error:'), log a warning to stderr and return empty array.
       - Call `parseProductIdeationOutput(result.stdout)` to parse and validate items.
       - If parsed items array is empty, log a warning and return empty array.
       - Return the parsed items.
