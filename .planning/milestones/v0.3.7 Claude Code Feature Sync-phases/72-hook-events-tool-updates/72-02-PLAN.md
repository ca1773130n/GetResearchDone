<objective>
Add ExitWorktree tool calls to GRD's worktree completion flow for native isolation mode. Currently, when using Claude Code-managed worktrees (isolation_mode=native), the executor writes artifacts and the orchestrator presents completion options, but neither explicitly exits the worktree. The ExitWorktree tool (added in Claude Code v2.1.72) should be called to properly leave the worktree before the completion flow.

Purpose: Without ExitWorktree, the agent may remain in the worktree context after execution completes, which can cause confusion during the completion flow (merge/PR/keep/discard). Calling ExitWorktree ensures the agent is back in the main repository before presenting completion options.

Output: Updated execute-phase.md and grd-executor.md with ExitWorktree calls in the native isolation path, plus CLAUDE_SKILL_DIR documentation comments.

Requirements: REQ-96, REQ-97
</objective>
