  <name>Task 1: Add backward-compatible fallback to paths.js</name>
  <files>lib/paths.js</files>
  <action>
    Modify `lib/paths.js` (created by Phase 32) to add backward compatibility for the transition period. The core issue: `phasesDir(cwd)` currently returns `.planning/milestones/{milestone}/phases/` but the actual directory on disk is still `.planning/phases/` (Phase 35 performs the physical migration). Tests create fixture directories at `.planning/phases/` and will break if the code suddenly looks in `.planning/milestones/anonymous/phases/`.

    **Strategy: Filesystem-aware fallback.** For each path function that accepts a milestone parameter (`phasesDir`, `phaseDir`, `researchDir`, `codebaseDir`, `todosDir`), add fallback logic:

    1. Compute the new-style path: `path.join(cwd, '.planning', 'milestones', milestone, 'phases')`
    2. If the new-style path's PARENT directory exists (i.e., `.planning/milestones/{milestone}/` exists), return the new-style path
    3. Otherwise, return the OLD-style path: `path.join(cwd, '.planning', 'phases')`

    Implementation pattern for each function:
    ```javascript
    function phasesDir(cwd, milestone) {
      const ms = milestone != null ? milestone : currentMilestone(cwd);
      const newPath = path.join(cwd, '.planning', 'milestones', ms, 'phases');
      const milestoneRoot = path.join(cwd, '.planning', 'milestones', ms);
      // Backward compat: if the milestone directory doesn't exist yet, fall back to old layout
      try {
        if (fs.existsSync(milestoneRoot)) return newPath;
      } catch {}
      return path.join(cwd, '.planning', 'phases');
    }
    ```

    Apply the same pattern to: `researchDir`, `codebaseDir`, `todosDir`.

    For `phaseDir(cwd, milestone, phaseDirName)`: use the result of `phasesDir(cwd, milestone)` + phaseDirName.

    For `quickDir(cwd)`: check if `.planning/milestones/anonymous/` exists; fall back to `.planning/quick`.

    `milestonesDir(cwd)` and `archivedPhasesDir(cwd, version)` do NOT need fallback (they already resolve to `.planning/milestones/` which exists from Phase 26).

    **IMPORTANT:** Do NOT change the function signatures. Do NOT change the exports. Only add internal fallback logic. The existing tests for `paths.js` may need minor updates to account for the fallback (create milestone directories in fixtures where the new-style path is expected, or accept that fallback returns old-style paths). If paths.test.js tests fail because they expected new-style paths but get old-style paths (fixture dirs don't have `.planning/milestones/{milestone}/` created), update those specific test cases to either (a) create the milestone directory in the fixture, or (b) assert the fallback old-style path.

    After implementation, run:
    ```bash
    npx jest tests/unit/paths.test.js
    npm test
    ```
