       ```js
       milestone_branch: config.branching_strategy === 'milestone'
         ? (config.milestone_branch_template || 'grd/{milestone}-{slug}')
             .replace('{milestone}', milestone.version)
             .replace('{slug}', generateSlugInternal(milestone.name) || 'milestone')
         : null,
       ```

    5. **Update worktree_path computation** to use project-local directory instead of `os.tmpdir()`:
       Replace the existing `worktree_path` computation:
       ```js
       // OLD:

         ? path.join(
             fs.realpathSync(os.tmpdir()),
             `grd-worktree-${milestone.version}-${phaseInfo.phase_number}`
           )
         : null,
       ```
       With:
       ```js
       // NEW:
       worktree_path: phaseInfo
         ? path.resolve(cwd, config.worktree_dir || '.worktrees/', `${milestone.version}-${phaseInfo.phase_number}`)
         : null,
       ```

    6. **Update branch_name computation** for milestone strategy:
       The existing code computes `branch_name` as the milestone branch when `branching_strategy === 'milestone'`. This should actually be the PHASE branch (since phase worktrees always use phase branches regardless of strategy). Fix:
       ```js
       branch_name:
         config.branching_strategy !== 'none' && phaseInfo
           ? config.phase_branch_template
               .replace('{milestone}', milestone.version)
               .replace('{phase}', phaseInfo.phase_number)
               .replace('{slug}', phaseInfo.phase_slug || 'phase')
           : null,
       ```
       This ensures `branch_name` is always the phase branch name for both `phase` and `milestone` strategies. The distinction between strategies is captured by `target_branch` (where the PR goes).

    7. **Remove the `os` import** if it was only used for `os.tmpdir()` in the worktree_path computation. Check if `os` is used elsewhere in the file first -- it IS used in the existing code for `os.tmpdir()`. After the change, check if any remaining references exist. If not, remove `os` from the destructured imports at the top.
