# Phase 37 Plan 01: Fix quickDir Routing + Migration Skill

## Objective

Fix three bugs shipped with v0.2.1:
1. `quickDir()` always routes to `anonymous` instead of the current milestone
2. `cmdMigrateDirs` hardcodes `quick/` target to `anonymous`
3. No `/grd:migrate` skill exists for users to invoke migration

## Tasks

### Task 1: Fix quickDir() in lib/paths.js
type="auto"

Add optional `milestone` parameter to `quickDir()` matching the pattern used by all other directory functions (`phasesDir`, `researchDir`, `todosDir`, `codebaseDir`). When omitted, call `currentMilestone(cwd)` instead of hardcoding `'anonymous'`.

**Done criteria:** `quickDir(cwd)` returns milestone-scoped path for current milestone; `quickDir(cwd, 'v1.0')` returns path for explicit milestone; backward-compatible fallback still works.

### Task 2: Update quickDir tests in tests/unit/paths.test.js
type="auto"

Rewrite `describe('quickDir')` block:
- "falls back to old-style when milestone dir does not exist on disk"
- "returns new-style when milestone dir exists on disk" (test with v0.2.1 milestone, not anonymous)
- "accepts an optional milestone parameter"
- "uses anonymous when currentMilestone returns anonymous"

**Done criteria:** 4 tests pass covering all quickDir scenarios.

### Task 3: Fix cmdMigrateDirs quick routing in lib/commands.js
type="auto"

Change `{ name: 'quick', target: 'anonymous' }` to `{ name: 'quick', target: milestone }`.

**Done criteria:** quick/ migrates to `milestones/{current_milestone}/quick/`.

### Task 4: Update cmdMigrateDirs test in tests/unit/commands.test.js
type="auto"

Update test to assert quick/ goes to `milestones/v1.0/quick/` (fixture's milestone) not `anonymous`. Assert `milestones/anonymous/quick/` does NOT exist.

**Done criteria:** All 10 cmdMigrateDirs tests pass.

### Task 5: Create /grd:migrate skill (commands/migrate.md)
type="auto"

Create skill that scans `.planning/` for migration needs, classifies items as trivial vs complex, runs `migrate-dirs` CLI for trivial items, and spawns `grd-migrator` agent for complex items.

**Done criteria:** File exists with proper frontmatter and process steps.

### Task 6: Create grd-migrator agent (agents/grd-migrator.md)
type="auto"

Create agent that handles flat milestone files, legacy phase dirs, orphan docs, and non-standard files.

**Done criteria:** File exists with proper frontmatter, migration rules, and execution flow.
