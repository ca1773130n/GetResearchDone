### Phase 26: Phase Directory Collision Fix

**Goal:** Prevent phase directory collisions across milestones by archiving phase directories during milestone completion and adding a pre-flight validation gate system.

**Verification level:** 2 (proxy)

| Plan | Title | Status |
|------|-------|--------|
| 26-01 | Phase directory archival and validation gate system | Complete |

#### What was delivered

1. **Phase directory archival** — `cmdMilestoneComplete` archives all phase directories to `.planning/milestones/{version}-phases/` and removes originals from `.planning/phases/`
2. **Validation gate system** (`lib/gates.js`) — 6 gate checks, declarative registry mapping 10 commands, YOLO bypass, new-project safety
3. **Gate integration** — Pre-flight gates in `cmdInitExecutePhase`, `cmdInitPlanPhase`, `cmdInitNewMilestone`, `cmdPhaseAdd`, `cmdPhaseInsert`, `cmdPhaseComplete`, `cmdMilestoneComplete`
4. **Config enhancements** — `autonomous_mode` in `loadConfig()`, `consistency_warning` in `findPhaseInternal`, `suggested_start_phase` in milestone init
5. **Tests** — 34 new tests (1,433 total), covering all gate checks, archival, and config additions
