  <name>Task 2: GREEN -- Implement refinement and promotion functions in lib/long-term-roadmap.js</name>
  <files>lib/long-term-roadmap.js</files>
  <action>
    Add 4 new exported functions to `lib/long-term-roadmap.js`. These operate on raw markdown content strings (not parsed objects) so they preserve formatting and sections they don't modify.

    **Function 1: `getMilestoneTier(content, milestoneVersion)`**

    Determine which tier (now/next/later) a milestone version belongs to.

    1. Parse content with `parseLongTermRoadmap(content)`.
    2. If `parsed.now && parsed.now.version === milestoneVersion` return `'now'`.
    3. If any `parsed.next[i].version === milestoneVersion` return `'next'`.
    4. If any `parsed.later[i].version === milestoneVersion` return `'later'`.
    5. Return `null` if not found.

    **Function 2: `refineMilestone(content, milestoneVersion, updates)`**

    Apply partial updates to a milestone's markdown section in-place.

    `updates` is an object with optional keys: `goal`, `success_criteria` (array), `rough_phase_sketch` (array), `open_questions` (array), `open_research_questions` (array).

    1. Call `getMilestoneTier(content, milestoneVersion)` to find the tier.
    2. If tier is null, return `{ error: 'Milestone ' + milestoneVersion + ' not found' }`.
    3. Find the milestone's section in the raw content using regex:
       - For Now: locate `## Current Milestone (Now)` section
       - For Next/Later: locate the `### {version} - {name}` subsection heading
    4. For each key in `updates`:
       - `goal`: Find the `#### Goal` (or `### Goal` for Now) heading within the milestone section. Replace the paragraph following it with the new goal text.
       - `success_criteria`: Find `#### Success Criteria` (or `### Success Criteria` for Now) heading. Replace the bullet list with new items (format: `- {item}`).
       - `rough_phase_sketch`: Find `#### Rough Phase Sketch` heading. Replace numbered list with new items (format: `{N}. {item}`).
       - `open_questions`: Find `#### Open Questions` heading. Replace bullet list with new items.
       - `open_research_questions`: Find `#### Open Research Questions` heading. Replace bullet list with new items.
    5. Return the updated content string.

    **Implementation approach for in-place section replacement:**
    - Locate the milestone section boundaries (start heading to next same-level heading)
    - Within that section, locate the subsection to replace (heading to next heading or section end)
    - Build replacement text and splice it into the content
    - Use helper function `replaceSubsection(sectionText, headingPattern, newContent)` that finds a heading within section text and replaces content after it until the next heading

    **Function 3: `promoteMilestone(content, milestoneVersion)`**

    Move a milestone one tier up: Later->Next or Next->Now.

    1. Call `getMilestoneTier(content, milestoneVersion)` to find current tier.
    2. If tier is `null`, return `{ error: 'Milestone ' + milestoneVersion + ' not found' }`.
    3. If tier is `'now'`, return `{ error: 'Milestone ' + milestoneVersion + ' is already in Now tier and cannot be promoted further' }`.
    4. Parse the content with `parseLongTermRoadmap(content)`.

    **Later -> Next promotion:**
    5a. Find the milestone in `parsed.later` by version.
    5b. Remove the milestone's `### {version} - {name}` subsection from the `## Later Milestones` section in raw content.
    5c. Build a new Next-tier subsection for this milestone:
        - Copy over: version, name, goal, success_criteria, dependencies
        - Add required Next fields with placeholders: `**Estimated Start:** TBD`, `**Estimated Duration:** TBD`
        - Add empty `#### Rough Phase Sketch` and `#### Open Questions` sections
        - Preserve `success_criteria` if they existed in Later
    5d. Append the new subsection to the `## Next Milestones` section in raw content.
    5e. Return updated content string.

    **Next -> Now promotion:**
    6a. Find the milestone in `parsed.next` by version.
    6b. Remove the milestone's `### {version} - {name}` subsection from the `## Next Milestones` section.
    6c. Build a new Now section using the milestone data:
        - `**Milestone:** {version} - {name}`
        - `**Status:** In Progress`
        - `**Start:** {today's date}` (or estimated_start if available)
        - `### Goal` with the milestone's goal
        - `### Success Criteria` with the milestone's success_criteria
    6d. Replace the entire `## Current Milestone (Now)` section content with the new Now section.
    6e. Return updated content string.

    **Function 4: `updateRefinementHistory(content, action, details)`**

    Append a row to the Refinement History markdown table.

    1. Find the `## Refinement History` section in content.
    2. Find the last table row (line starting with `|` that is not the header or separator).
    3. After the last row, insert: `| {today's date} | {action} | {details} |`.
    4. Return updated content string.
    5. If no Refinement History section exists, return content unchanged (defensive).
