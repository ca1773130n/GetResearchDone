<task type="auto">
  <name>Task 2: GREEN — Implement doc drift detection functions and wire into runQualityAnalysis</name>
  <files>lib/cleanup.js</files>
  <action>
    Add three new functions to lib/cleanup.js and update runQualityAnalysis:

    **1. analyzeChangelogDrift(cwd)**
    - Find CHANGELOG.md at project root (path.join(cwd, 'CHANGELOG.md'))
    - If not found, return [] (graceful skip)
    - Get CHANGELOG.md mtime via fs.statSync
    - Scan .planning/phases/*/ for all *-SUMMARY.md files
    - If no SUMMARY.md files exist, return [] (nothing to compare)
    - Find the newest SUMMARY.md mtime
    - If CHANGELOG.md mtime < newest SUMMARY.md mtime, return [{file: 'CHANGELOG.md', reason: 'Not updated since last plan completion', last_modified: changelogMtime.toISOString(), latest_summary: newestSummaryMtime.toISOString()}]
    - Otherwise return []

    **2. analyzeReadmeLinks(cwd)**
    - Find README.md at project root (path.join(cwd, 'README.md'))
    - If not found, return [] (graceful skip)
    - Read content, find all markdown links via regex: /\[([^\]]*)\]\(([^)"\s]+)(?:\s+"[^"]*")?\)/g
    - For each match, extract the path (group 2)
    - Skip external links: if path starts with 'http://' or 'https://' or '#', skip
    - For remaining (relative) paths, resolve against cwd: path.resolve(cwd, linkPath)
    - If resolved path does not exist (fs.existsSync), add to broken list: {file: 'README.md', link: linkPath, line: lineNumber}
    - Return broken links array

    **3. analyzeJsdocDrift(cwd, files)**
    - If files is empty, return []
    - For each file:
      - Read content
      - Find all JSDoc blocks + their following function declarations using regex
      - JSDoc block: /\/\*\*[\s\S]*?\*\//g
      - Extract @param names from JSDoc: /@param\s+(?:\{[^}]*\}\s+)?(\w+)/g
      - Find the function declaration immediately after the JSDoc block (within next 3 lines)
      - Function patterns to match:
        - `function name(param1, param2)` — standard function
        - `const name = (param1, param2) =>` — arrow function
        - `name(param1, param2) {` — method shorthand
      - Extract actual parameter names from the function signature
      - Compare: for each @param name not in actual params, report as "extra @param"
      - Compare: for each actual param not in @param annotations, report as "missing @param"
      - Return [{file, line, functionName, issue: 'extra @param: xyz' | 'missing @param: xyz'}]
    - Avoid false positives: skip destructured params ({a, b}), rest params (...args), and default values (x = 1) — extract base name only

    **4. Update runQualityAnalysis(cwd, phaseNum)**
    - After existing 3 analyses, check if config.doc_sync is true
    - If true, run all 3 doc drift functions: analyzeChangelogDrift(cwd), analyzeReadmeLinks(cwd), analyzeJsdocDrift(cwd, jsFiles)
    - Add to summary: doc_drift_issues (sum of all 3 results lengths)
    - Add to details: doc_drift: { changelog: changelogResults, readme_links: readmeResults, jsdoc: jsdocResults }
    - Update total_issues to include doc_drift_issues
    - If config.doc_sync is false, omit doc_drift from both summary and details (backward compatible)

    **5. Update module.exports**
    - Add analyzeChangelogDrift, analyzeReadmeLinks, analyzeJsdocDrift to exports

    - Same JSDoc comment style on exported functions
    - Same result object shape conventions
