    **1. Modify `cmdPhaseDetail` in `lib/commands.js` to extract and display requirements:**

    After the existing code that reads STATE.md for decisions (around line 1370), add requirement extraction logic:

    **Step A: Read ROADMAP.md and extract Requirements field for this phase:**
    - Read `.planning/ROADMAP.md` via `safeReadFile`
    - Find the phase section matching the current phase number (look for `### Phase {phaseNumber}:` heading)
    - Within that section, find `**Requirements**: REQ-XX, REQ-YY` line
    - Parse the comma-separated REQ-IDs into an array (trim whitespace)
    - If no Requirements line found, set `reqIds = []`

    **Step B: Look up each requirement's details:**
    - If `reqIds` is non-empty:
      - Read `.planning/REQUIREMENTS.md` via `safeReadFile`
      - Call `parseRequirements(reqContent)` and `parseTraceabilityMatrix(reqContent)` (functions from Plan 01, already in same file)
      - For each REQ-ID in the list:
        - Find matching requirement in parsed requirements
        - Find matching status row in traceability matrix
        - Build requirement summary: `{ id, title, priority, status }`
        - If requirement not found (e.g., typo in ROADMAP), include it with `title: null, priority: null, status: "Unknown"`
    - Store as `phaseRequirements` array

    **Step C: Add requirements to JSON result:**
    - Add `requirements: phaseRequirements` to the result object (alongside plans, decisions, artifacts)

    **Step D: Add requirements to TUI rendering:**
    - After the Artifacts section (around line 1441), add requirements rendering:
    ```javascript
    if (phaseRequirements.length > 0) {
      tui += `\n## Requirements (${phaseRequirements.length})\n`;
      tui += '| REQ | Title | Priority | Status |\n';
      tui += '|-----|-------|----------|--------|\n';
      for (const r of phaseRequirements) {
        const title = r.title || '-';
        const truncTitle = title.length > 40 ? title.substring(0, 37) + '...' : title;
        tui += `| ${r.id} | ${truncTitle} | ${r.priority || '-'} | ${r.status || '-'} |\n`;
      }
    }
    ```

    **Important implementation notes:**
    - The ROADMAP.md format for requirements is: `**Requirements**: REQ-31, REQ-32, REQ-33, REQ-34` (on a line inside the phase section). Use regex: `/\*\*Requirements\*\*:\s*(.+)/i`
    - Phase section boundaries: starts with `### Phase {N}:` and ends at next `### Phase` or `##` heading
    - `parseRequirements` and `parseTraceabilityMatrix` are already defined in this file from Plan 01 — call them directly (no import needed)
    - Handle missing REQUIREMENTS.md gracefully (return empty requirements array)
