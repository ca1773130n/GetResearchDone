    **1. Add requirement parsing helper functions to `lib/commands.js`:**

    Add a new section after the existing `// ─── Health ──` section (before Exports) with header `// ─── Requirements ──`:

    **`parseRequirements(content)`** — Parse a REQUIREMENTS.md file content string into an array of requirement objects:
    - Split content by `### REQ-` headings
    - For each heading, extract:
      - `id`: The REQ-NN identifier (e.g., "REQ-31")
      - `title`: Text after "REQ-NN: " on the heading line
      - `priority`: Value after `**Priority:**` (e.g., "P0")
      - `category`: Value after `**Category:**` (e.g., "CLI")
      - `deferred_from`: Value after `**Deferred from:**` if present, else null
      - `resolves`: Value after `**Resolves:**` if present, else null
      - `description`: Remaining text after metadata fields, trimmed
    - Return array of requirement objects
    - Handle edge cases: missing fields gracefully default to null

    **`parseTraceabilityMatrix(content)`** — Parse the Traceability Matrix table from REQUIREMENTS.md:
    - Find section starting with `## Traceability Matrix`
    - Parse the markdown table rows (skip header row and separator row)
    - For each data row, extract: req, feature, priority, phase, status
    - Return array of objects
    - Handle edge cases: missing table, empty table

    **2. Add command functions to `lib/commands.js`:**

    **`cmdRequirementGet(cwd, reqId, raw)`:**
    - Read `.planning/REQUIREMENTS.md` via `safeReadFile`
    - Call `parseRequirements()` on content
    - Find matching requirement by id (case-insensitive match)
    - If found, also look up status from traceability matrix via `parseTraceabilityMatrix()`
    - Merge status and phase from traceability into the requirement object
    - If NOT found in current file, scan `.planning/milestones/v*-REQUIREMENTS.md` files:
      - Read each file, parse requirements, search for the ID
      - If found in archived file, include `milestone` field indicating source
    - If still not found, output error JSON: `{ error: "Requirement not found", id: reqId }`
    - Output JSON result via `output(result, raw)`

    **`cmdRequirementList(cwd, filters, raw)`:**
    - `filters` is an object: `{ phase, priority, status, category, all }`
    - Read `.planning/REQUIREMENTS.md`, parse requirements and traceability matrix
    - Merge status/phase from matrix into each requirement
    - If `filters.all` is true, also read all `.planning/milestones/v*-REQUIREMENTS.md` files and merge their requirements (with `milestone` field)
    - Apply filters (AND logic):
      - `phase`: Match against traceability matrix phase column (e.g., "Phase 19" matches when filter is "19")
      - `priority`: Exact match (e.g., "P0")
      - `status`: Exact match (e.g., "Pending", "Done")
      - `category`: Exact match (e.g., "CLI")
    - Output JSON: `{ requirements: [...], count: N, filters_applied: {...} }`

    **`cmdRequirementTraceability(cwd, filters, raw)`:**
    - `filters` is an object: `{ phase }`
    - Read `.planning/REQUIREMENTS.md`, parse traceability matrix
    - If `filters.phase` specified, filter rows where phase column contains that phase number
    - Output JSON: `{ matrix: [...], count: N }`

    **3. Export new functions from `lib/commands.js`:**

    Add to the `module.exports` object:
    - `cmdRequirementGet`
    - `cmdRequirementList`
    - `cmdRequirementTraceability`
    - `parseRequirements` (exported for reuse by cmdPhaseDetail in Plan 02 and for testing)
    - `parseTraceabilityMatrix` (exported for reuse and testing)

    **4. Add CLI routing in `bin/grd-tools.js`:**

    Add `REQUIREMENT_SUBS = ['get', 'list', 'traceability']` constant alongside other SUB arrays.

    Add a `case 'requirement':` block in the switch statement (before `default:`):
    ```javascript
    case 'requirement': {
      const sub = args[1];
      validateSubcommand(sub, REQUIREMENT_SUBS, 'requirement');
      if (sub === 'get') {
        if (!args[2]) error('REQ-ID required');
        cmdRequirementGet(cwd, args[2], raw);
      } else if (sub === 'list') {
        cmdRequirementList(cwd, {
          phase: flag(args, '--phase', null),
          priority: flag(args, '--priority', null),
          status: flag(args, '--status', null),
          category: flag(args, '--category', null),
          all: args.includes('--all'),
        }, raw);
      } else if (sub === 'traceability') {
        cmdRequirementTraceability(cwd, {
          phase: flag(args, '--phase', null),
        }, raw);
      }
      break;
    }
    ```

    Import the new functions in the require block at top of `bin/grd-tools.js` (add to existing destructured import from `../lib/commands`).
