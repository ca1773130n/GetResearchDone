    Create `lib/long-term-roadmap.js` implementing all functions tested in Task 1.

    **Module structure:**

    ```js
    const fs = require('fs');
    const path = require('path');
    const { safeReadFile, output, error } = require('./utils');
    const { extractFrontmatter } = require('./frontmatter');
    ```

    **Function 1: `parseLongTermRoadmap(content)`**

    Parse LONG-TERM-ROADMAP.md content string into structured object.

    1. Use `extractFrontmatter(content)` to get YAML frontmatter fields (`project`, `roadmap_type`, `created`, `last_refined`, `planning_horizon`).
    2. Parse `## Current Milestone (Now)` section:
       - Extract `**Milestone:**` (name), find version pattern `v\d+\.\d+` within it
       - Extract `**Status:**`, `**Start:**`, `**Target:**`
       - Extract `### Goal` section text (paragraph after heading)
       - Extract `### Success Criteria` section (bullet list items)
       - Extract `### Open Questions` section (bullet list items)
    3. Parse `## Next Milestones` section:
       - Find all `### vX.Y.Z - Name` subsections within
       - For each: extract `**Status:**`, `**Estimated Start:**`, `**Estimated Duration:**`, `**Dependencies:**`
       - Extract `#### Goal`, `#### Success Criteria`, `#### Rough Phase Sketch` (numbered list), `#### Open Questions`
    4. Parse `## Later Milestones` section:
       - Find all `### vX.Y.Z - Name` subsections within
       - For each: extract `**Status:**`, `**Estimated Timeline:**`, `**Dependencies:**`
       - Extract `#### Goal`, `#### Success Criteria`, `#### Open Research Questions`
    5. Parse `## Refinement History` table if present.
    6. Return: `{ frontmatter: {...}, now: {...}, next: [...], later: [...], refinement_history: [...] }`

    **Implementation notes for section parsing:**
    - Use regex to find tier headings: `/^## Current Milestone \(Now\)/m`, `/^## Next Milestones/m`, `/^## Later Milestones/m`
    - Slice content between tier headings to isolate each section
    - Within each section, use `### ` to delimit individual milestones
    - For extracting field values: `/\*\*Milestone:\*\*\s*(.+)/`, `/\*\*Status:\*\*\s*(.+)/`, etc.
    - For bullet lists: match lines starting with `- ` after the section heading
    - For numbered lists (Rough Phase Sketch): match lines starting with `\d+\. `
    - Extract version from milestone heading: `/v(\d+\.\d+\.\d+)/` or `/v(\d+\.\d+)/`

    **Function 2: `validateLongTermRoadmap(parsed)`**

    Validate a parsed roadmap object against the schema.

    1. Check required fields in frontmatter: `project` must exist.
    2. Check `now` milestone: must have `goal` and `milestone` (name).
    3. Check each `next` milestone: must have `goal` and `milestone`.
    4. Check each `later` milestone: must have `goal`.
    5. Count total milestones in `next` + `later`: warn if > 5.
    6. Check for success_criteria: warn (not error) if Later milestones lack them.
    7. Return `{ valid: boolean, errors: string[], warnings: string[] }`.

    **Function 3: `getPlanningMode(cwd)`**

    Detect planning mode from filesystem.

    1. Check if `.planning/LONG-TERM-ROADMAP.md` exists at `cwd`.
    2. If not exists: return `'progressive'`.
    3. If exists: read file, extract frontmatter.
    4. If frontmatter has `roadmap_type: 'progressive'`: return `'progressive'` (explicit override).
    5. Otherwise: return `'hierarchical'`.

    **Function 4: `generateLongTermRoadmap(milestones, projectName, planningHorizon)`**

    Generate LONG-TERM-ROADMAP.md content from structured milestone data.

    `milestones` is an array where:
    - `milestones[0]` = Now milestone (detailed)
    - `milestones[1..2]` = Next milestones (refined)
    - `milestones[3+]` = Later milestones (rough)

    Each milestone object has: `{ name, version, goal, success_criteria, status, ... }` plus tier-specific fields.

    1. Build YAML frontmatter: `project`, `roadmap_type: "hierarchical"`, `created` (today), `last_refined` (today), `planning_horizon` (from param or default "6 months").
    2. Build `## Current Milestone (Now)` section from milestones[0].
    3. Build `## Next Milestones` section from milestones[1..2] (or however many are Next tier).
    4. Build `## Later Milestones` from remaining milestones.
    5. Build `## Milestone Dependency Graph` with simple text diagram.
    6. Build `## Refinement History` table with initial creation entry.
    7. Return complete markdown string.

    **Function 5: `formatLongTermRoadmap(parsed)`**

    Format parsed roadmap for human-readable display.

    1. Header: `# Long-Term Roadmap: {project}` with planning horizon info.
    2. Now section: `[Now] {version} - {name}  Status: {status}  {goal}`
    3. Next section: For each: `[Next] {version} - {name}  Status: {status}  {goal}`
    4. Later section: For each: `[Later] {version} - {name}  Timeline: {timeline}  {goal}`
    5. Include dependency graph and refinement summary.
    6. Return formatted string.
