<objective>
Fix BUG-48-002 (roadmap goal regex mismatch) and BUG-48-003 (state-snapshot field name mismatch).

BUG-48-002: `cmdRoadmapGetPhase()` and `analyzeRoadmap()` in lib/roadmap.js use regex `/\*\*Goal:\*\*\s*([^\n]+)/i` which expects `**Goal:**` but ROADMAP.md uses `**Goal**:` (colon outside bold). Fix both regex patterns to handle both formats.

BUG-48-003: `cmdStateSnapshot()` in lib/state.js looks for `**Current Phase:**` but STATE.md uses `**Active phase:**`. Fix the field name lookup to match the actual STATE.md field names.
</objective>

<execution_context>
- **Codebase convention**: `**Goal:**` (colon inside bold) is the regex expectation, `**Goal**:` (colon outside bold) is the actual ROADMAP.md format. Both should be supported.
- **Existing pattern**: The `depends_on` regex in analyzeRoadmap already handles both formats with `/\*\*Depends on:?\*\*:?\s*/i`. Apply the same pattern to goal.
- **STATE.md format**: Uses `**Active phase:**` not `**Current Phase:**`. Also uses `**Status:**` and `**Progress:**` which already work because their names match.
</execution_context>

## Task 1: Fix goal regex in lib/roadmap.js (BUG-48-002)

In `cmdRoadmapGetPhase()` (line ~214) and `analyzeRoadmap()` (line ~347), change the goal regex from:
```js
/\*\*Goal:\*\*\s*([^\n]+)/i
```
to:
```js
/\*\*Goal:?\*\*:?\s*([^\n]+)/i
```

This handles both `**Goal:**` and `**Goal**:` formats (same pattern used for `depends_on`).

## Task 2: Fix field name mismatches in cmdStateSnapshot (BUG-48-003)

In `cmdStateSnapshot()` in lib/state.js (line ~585), the function uses `extractField('Current Phase')` but STATE.md uses `**Active phase:**`. Fix the field lookups:

1. `extractField('Current Phase')` -> try `'Active phase'` first, fall back to `'Current Phase'`
2. `extractField('Current Phase Name')` -> this field does not exist in STATE.md; extract the phase name from the "Active phase" value (format: "Phase N of M (Name)")
3. `extractField('Total Phases')` -> extract from the "Active phase" value

The STATE.md format for the current phase line is:
```
- **Active phase:** Phase 49 of 52 (Bug Discovery & Fixes)
```

So we need to parse this single line to extract `current_phase`, `current_phase_name`, and `total_phases`.
