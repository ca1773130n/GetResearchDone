<execution_context>
- **Plan format**: Plans use `<objective>...</objective>` XML tags in body, not frontmatter `objective` key.
- **Frontmatter key names**: The YAML parser in lib/frontmatter.js preserves underscores. Keys are `files_modified`, not `files-modified`.
- **STATE.md field format**: Uses `**Field Name:**` with spaces. CLI args use `--field_name` with underscores.
</execution_context>

## Task 1: Fix objective extraction in cmdPhasePlanIndex (BUG-48-004)

In `cmdPhasePlanIndex()` in lib/commands.js (line ~705), the objective is set from `fm.objective`. Since plans use `<objective>` XML tags in the body, add extraction from the body:

```js
// Extract objective from <objective> tag if not in frontmatter
let objective = fm.objective || null;
if (!objective) {
  const objMatch = content.match(/<objective>\s*([\s\S]*?)\s*<\/objective>/i);
  if (objMatch) {
    // Take first line/sentence as the objective summary
    objective = objMatch[1].trim().split('\n')[0].trim();
  }
}
```

## Task 2: Fix files_modified key name in cmdPhasePlanIndex (BUG-48-004)

In `cmdPhasePlanIndex()` in lib/commands.js (line ~690), the code checks `fm['files-modified']` (hyphen). The frontmatter parser preserves underscores, so the key is `files_modified`. Fix to check both:

```js
let filesModified = [];
if (fm.files_modified || fm['files-modified']) {
  const raw = fm.files_modified || fm['files-modified'];
  filesModified = Array.isArray(raw) ? raw : [raw];
}
```
