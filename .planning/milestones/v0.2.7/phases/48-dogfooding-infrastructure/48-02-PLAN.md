<task type="auto">
  <name>Task 1: Create test-grd.sh local CLI testing script</name>
  <files>testbed/test-grd.sh</files>
  <action>
    Create a shell script at testbed/test-grd.sh that serves as the local CLI testing harness:

    ```bash
    #!/usr/bin/env bash
    # test-grd.sh — Run GRD CLI commands against this testbed using local source
    # Usage: ./test-grd.sh <command> [args...]
    # Examples:
    #   ./test-grd.sh state load
    #   ./test-grd.sh roadmap get-phase 1
    #   ./test-grd.sh validate consistency
    #   ./test-grd.sh phase-plan-index 1
    ```

    The script must:
    1. Resolve the path to local bin/grd-tools.js relative to the testbed directory:
       - `GRD_CLI="$(dirname "$0")/../bin/grd-tools.js"` (one level up from testbed/)
    2. Verify the CLI file exists, exit with error if not
    3. Execute: `node "$GRD_CLI" "$@"` (passes all arguments through)
    4. Return the exit code from node

    Additionally, create a `test-grd-validate.sh` companion script that runs a battery of validation commands:
    1. `state load` — must return valid JSON
    2. `state get current_phase` — must return a value
    3. `roadmap get-phase 1` — must find the phase
    4. `validate consistency` — must pass
    5. `state-snapshot` — must return structured JSON
    6. `phase-plan-index 1` — must work (even if no plans yet)

    Each validation step should print PASS/FAIL and a summary at the end.

    Make both scripts executable (chmod +x).

    IMPORTANT: The relative path `../bin/grd-tools.js` is critical — this is what makes it "local CLI testing" rather than using the cached plugin. This path works because testbed/ is a subdirectory of the GRD project root.
  </action>
  <verify>
    test -x testbed/test-grd.sh confirms script is executable
    head -1 testbed/test-grd.sh | grep -q "bash" confirms it's a bash script
    grep -q "bin/grd-tools.js" testbed/test-grd.sh confirms it references local CLI
    (Level 1: Sanity)
  </verify>
  <done>test-grd.sh exists, is executable, and correctly references local bin/grd-tools.js via relative path</done>
