## Task 1: Consolidate cmdInit* boilerplate in lib/context.js

The 14 `cmdInit*` functions in context.js share a common pattern:
1. Read config, roadmap, state
2. Check path existence
3. Build a result object with `project_root`, `milestone`, `config`, etc.

Extract a shared internal helper:

```js
function buildBaseInitContext(cwd) {
  const config = loadConfig(cwd);
  const milestone = getMilestoneInfo(cwd);
  const planningDir = path.join(cwd, '.planning');
  const milestoneVersion = currentMilestone(cwd);

  return {
    project_root: cwd,
    planning_dir: planningDir,
    milestone_version: milestoneVersion,
    milestone_name: milestone.name,
    config,
  };
}
```

Then use it in each `cmdInit*` function to reduce the repeated config+milestone+path setup. This should save 5-8 lines per function across 10+ functions = ~60-80 lines net reduction.
