# Plan 26-01: Phase Directory Collision Fix

## Goal

Prevent phase directory collisions across milestones by archiving phase directories during milestone completion and adding a pre-flight validation gate system.

## Context

Phase directories in `.planning/phases/` are flat (e.g., `05-background-agent-monitoring/`) with no milestone namespace. When a milestone completes and a new one starts, old phase directories persist. If the new milestone reuses phase numbers, `findPhaseInternal` matches old directories, causing stale plans to surface.

## Tasks

### Task 1: Phase Directory Archival in Milestone Complete
- **Files**: `lib/phase.js`, `commands/complete-milestone.md`
- Archive all phase directories to `.planning/milestones/{version}-phases/` during `cmdMilestoneComplete`
- Update result object with `archived.phases` and `archived.phase_count`
- Update command documentation with new success criteria

### Task 2: Validation Gate System
- **Files**: `lib/gates.js` (NEW), `lib/utils.js`, `lib/context.js`, `lib/phase.js`, `commands/new-milestone.md`
- Create `lib/gates.js` with 6 gate check functions and declarative gate registry
- Add `autonomous_mode` to `loadConfig()` in `lib/utils.js`
- Add `consistency_warning` to `findPhaseInternal` in `lib/utils.js`
- Integrate gates into `cmdInitExecutePhase`, `cmdInitPlanPhase`, `cmdInitNewMilestone` in `lib/context.js`
- Integrate gates into `cmdPhaseAdd`, `cmdPhaseInsert`, `cmdPhaseComplete`, `cmdMilestoneComplete` in `lib/phase.js`
- Refactor `cmdValidateConsistency` to reuse `checkOrphanedPhases` from gates

### Task 3: Tests
