    - Change `"name"` from `"grd"` to `"grd-tools"`
    - Keep `"version": "0.1.0"` (already matches VERSION file)
    - Remove `"private": true` line entirely
    - Add `"bin"` field:
      ```json
      "bin": {
        "grd-tools": "bin/grd-tools.js",
        "grd-mcp-server": "bin/grd-mcp-server.js"
      }
      ```
    - Add `"files"` whitelist:
      ```json
      "files": [
        "bin/",
        "lib/",
        "commands/",
        "agents/",
        ".claude-plugin/plugin.json"
      ]
      ```
    - Add `"postinstall"` script entry:
      ```json
      "postinstall": "node bin/postinstall.js"
      ```
    - Keep `"engines": { "node": ">=18" }` unchanged
    - Verify zero `"dependencies"` field (only `"devDependencies"` for jest/eslint/prettier)
    - Add `"repository"`, `"keywords"` fields for npm metadata:
      ```json
      "keywords": ["research", "workflow", "claude", "mcp", "cli"],
      "repository": { "type": "git", "url": "https://github.com/cameleon-x/grd" }
      ```

    **2. Create bin/postinstall.js:**

    - Add `#!/usr/bin/env node` shebang
    - Script creates the `.planning/` directory structure in the current working directory:
      - `.planning/`
      - `.planning/phases/`
      - `.planning/research/`
      - `.planning/research/deep-dives/`
      - `.planning/codebase/`
      - `.planning/todos/`
    - Creates default `.planning/config.json` with minimal config if not present:
      ```json
      {
        "model_profile": "balanced",
        "commit_docs": true,
        "autonomous_mode": false,
        "research_gates": {},
        "confirmation_gates": {},
        "eval_config": {
          "default_metrics": ["test_coverage_pct", "lint_error_count"],
          "baseline_tracking": true
        }
      }
      ```
    - **Idempotency:** Check `fs.existsSync()` before each `mkdirSync` and before writing config.json. If `.planning/` already exists with any content, do nothing and exit silently (exit code 0).
    - Use `fs.mkdirSync(dir, { recursive: true })` for directory creation.
    - Print a short message only when creating new structure: `"GRD: Created .planning/ directory structure"`.
    - If structure already exists, print nothing and exit 0.
    - Wrap entire script in try/catch. On error, print warning to stderr but exit 0 (postinstall must never fail and block npm install).
    - No external dependencies — only `fs` and `path` from Node.js built-ins.
  </action>
  <verify>

    2. `node bin/postinstall.js` in a temp directory creates .planning/ structure (Level 2: Proxy)
    3. Running `node bin/postinstall.js` again in same directory does nothing (idempotency check)
  </verify>
